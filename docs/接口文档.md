# QOJ API 文档

QOJ 后端提供 RESTful API 和 WebSocket 实时推送服务。所有 API 遵循统一响应格式，使用 JWT 进行身份认证。

---

## 基础信息

**Base URL**: `http://127.0.0.1:8080/api`

**API 版本**: v1

**认证方式**: JWT Bearer Token

**响应格式**: JSON

---

## 统一响应格式

所有 API 返回统一结构：

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    // 实际数据
  }
}
```

### 错误码

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 40000 | 请求参数错误 |
| 40001 | 未登录或登录已过期 |
| 40003 | 无权限 |
| 40004 | 资源不存在 |
| 40009 | 资源冲突 |
| 42900 | 请求过于频繁 |
| 50000 | 系统错误 |
| 50001 | 判题服务异常 |
| 50100 | 功能尚未实现 |

## 认证机制

### JWT 双 Token 设计

QOJ 使用 **Access Token** 和 **Refresh Token** 双 Token 机制：

- **Access Token**: 短期有效（默认 15 分钟），用于 API 请求认证
- **Refresh Token**: 长期有效（默认 7 天），用于刷新 Access Token

**流程**:
1. 用户登录 → 获取 Access Token + Refresh Token
2. 每次请求携带 Access Token（`Authorization: Bearer <token>`）
3. Access Token 过期 → 使用 Refresh Token 刷新
4. Refresh Token 过期 → 重新登录

### Token 轮换（Refresh Token Family）

每次刷新 Token 时：
- 旧 Refresh Token 失效并加入黑名单
- 生成新的 Access Token + Refresh Token（相同 `familyId`）
- 防止 Refresh Token 被盗用（重放攻击）

### 请求示例

```bash
# 携带 Access Token 请求
curl -H "Authorization: Bearer eyJhbGciOiJIUzUxMiJ9..." \
  http://127.0.0.1:8080/api/v1/problems
```

## 用户端 API (`/api/v1`)

### 1. 认证模块 (`/auth`)

#### 1.1 用户登录

**POST** `/api/v1/auth/login`

**请求体**:
```json
{
  "username": "student01",
  "password": "password123"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzUxMiJ9...",
    "refreshToken": "eyJhbGciOiJIUzUxMiJ9...",
    "user": {
      "id": 1,
      "username": "student01",
      "displayName": "张三",
      "role": "STUDENT",
      "studentNo": "2024001",
      "email": "student01@example.com"
    }
  }
}
```

#### 1.2 用户注册

**POST** `/api/v1/auth/register`

**请求体**:
```json
{
  "username": "newuser",
  "password": "password123",
  "email": "newuser@example.com",
  "displayName": "新用户",
  "studentNo": "2024999"
}
```

#### 1.3 刷新 Token

**POST** `/api/v1/auth/refresh`

**请求头**:
```
Authorization: Bearer <refresh_token>
```

**响应**:
```json
{
  "code": 200,
  "data": {
    "accessToken": "eyJhbGciOiJIUzUxMiJ9...",
    "refreshToken": "eyJhbGciOiJIUzUxMiJ9..."
  }
}
```

#### 1.4 退出登录

**POST** `/api/v1/auth/logout`

**功能**: 将当前 Token 加入黑名单，防止后续使用。

#### 1.5 获取当前用户信息

**GET** `/api/v1/auth/me`

**响应**:
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "student01",
    "displayName": "张三",
    "role": "STUDENT",
    "rating": 1523,
    "totalSolved": 42,
    "totalSubmissions": 187
  }
}
```

### 2. 题库模块 (`/problems`)

#### 2.1 获取题目列表

**GET** `/api/v1/problems`

**查询参数**:
- `page` (int): 页码，默认 1
- `size` (int): 每页数量，默认 20
- `difficulty` (string): 难度筛选（入门/简单/中等/困难/地狱）
- `tags` (string[]): 标签筛选，逗号分隔
- `keyword` (string): 关键词搜索（标题或题面）

**响应**:
```json
{
  "code": 200,
  "data": {
    "items": [
      {
        "id": 1,
        "title": "A+B Problem",
        "difficulty": "入门",
        "tags": ["数学", "模拟"],
        "acRate": 85.3,
        "timeLimit": 1000,
        "memoryLimit": 256,
        "score": 100,
        "attemptStatus": "AC"
      }
    ],
    "total": 123,
    "page": 1,
    "size": 20
  }
}
```

#### 2.2 获取题目详情

**GET** `/api/v1/problems/{id}`

**权限**: 公开题目无需登录；私有题目需为题目所有者或超级管理员。

**响应**:

## WebSocket 实时推送

QOJ 使用 STOMP 协议通过 WebSocket 推送实时消息。

**WebSocket URL**: `ws://127.0.0.1:8080/ws`

**协议**: STOMP over SockJS

### 连接认证

WebSocket 连接需携带 Access Token：

```javascript
import { Client } from '@stomp/stompjs';
import SockJS from 'sockjs-client';

const client = new Client({
  webSocketFactory: () => new SockJS('http://127.0.0.1:8080/ws'),
  connectHeaders: {
    Authorization: `Bearer ${accessToken}`
  },
  onConnect: () => {
    console.log('WebSocket 已连接');
  }
});

client.activate();
```

## 速率限制

为防止滥用，部分接口有速率限制：

| 接口 | 限制 |
|------|------|
| `/api/v1/auth/login` | 5 次/分钟 |
| `/api/v1/submissions` | 10 次/分钟 |
| `/api/v1/sandbox/run` | 3 次/分钟 |

超过限制返回错误码 **42900**（请求过于频繁）。

## 跨域配置

后端默认允许以下域名跨域访问：
- `http://127.0.0.1:5173`
- `http://localhost:5173`

生产环境需修改 `application.yml` 中的 `qoj.cors.allowed-origins`。

## 最佳实践

1. **Token 管理**: 使用 `localStorage` 存储 Token，在 API 请求前检查过期时间
2. **错误处理**: 统一处理 40001 错误（未登录），跳转到登录页
3. **WebSocket 重连**: 断线后自动重连，避免消息丢失
4. **分页加载**: 题目列表、提交列表等使用分页，避免一次加载过多数据
5. **防抖节流**: 搜索框输入等操作使用防抖，减少 API 请求
