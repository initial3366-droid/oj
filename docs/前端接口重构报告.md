## 🔧 核心模块详解

### 1. client.ts - 统一 API 客户端

**功能**:
- ✅ 自动带 `Authorization: Bearer ${token}`
- ✅ Token 自动获取和管理
- ✅ 401 自动跳转登录
- ✅ 403 权限不足错误
- ✅ 500 服务器错误统一处理
- ✅ 15 秒请求超时保护
- ✅ 统一解析 `ApiResponse<T>` 格式

**核心方法**:

```typescript
// GET 请求
export async function apiGet<T>(url: string, requireAuth = false): Promise<T>

// POST 请求
export async function apiPost<T>(url: string, body?: unknown, requireAuth = false): Promise<T>

// PUT 请求
export async function apiPut<T>(url: string, body?: unknown, requireAuth = false): Promise<T>

// DELETE 请求
export async function apiDelete<T>(url: string, requireAuth = false): Promise<T>

// Token 管理
export function setToken(token: string)
export function clearToken()
```

**错误处理逻辑**:

```typescript
// 401 未授权 - 自动清除 token 并跳转登录
if (response.status === 401) {
  clearToken();
  window.location.href = "/auth";
  throw new Error("未登录或登录已过期，请重新登录");
}

// 403 禁止访问
if (response.status === 403) {
  throw new Error(body?.message || "权限不足，无法访问该资源");
}

// 500 服务器错误
if (response.status === 500) {
  throw new Error(body?.message || "服务器内部错误，请稍后重试");
}
```

## ✅ 架构优势

### 1. 统一的请求处理

**Before**:
```typescript
// 每个页面都要写一遍
const token = window.localStorage.getItem("qoj.accessToken");
const response = await fetch(url, {
  headers: {
    Authorization: `Bearer ${token}`,
    "Content-Type": "application/json",
  }
});
const body = await response.json();
if (!response.ok || body.code !== 200) {
  // 手动处理错误
}
```

**After**:
```typescript
// 一行搞定
const data = await apiGet<T>(url, true);
```

### 2. 自动错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 401 Unauthorized | 自动清除 token，跳转登录页 |
| 403 Forbidden | 显示权限不足提示 |
| 500 Internal Server Error | 显示服务器错误提示 |
| Timeout | 15 秒超时，友好提示 |
| Network Error | 显示网络错误提示 |

### 3. 类型安全

```typescript
// ✅ 完全类型安全
```

## 📝 使用指南

### 1. 导入 API

```typescript
// 方式 1: 从模块导入
import { login, fetchMe } from "@/api/auth";
import { submitCode } from "@/api/submission";

// 方式 2: 从统一入口导入
import { login, fetchMe, submitCode } from "@/api";
```

### 2. 调用 API

```typescript
// 登录
try {
  const result = await login("username", "password");
  console.log("登录成功", result.accessToken);
} catch (error) {
  console.error("登录失败", error.message);
  // 401/403/500 等错误已自动处理
}

// 获取用户信息
try {
  const user = await fetchMe();
  console.log("用户信息", user);
} catch (error) {
  // 如果 401，已自动跳转登录页
  console.error("获取用户信息失败", error.message);
}

// 提交代码
try {
  const submission = await submitCode({
    problemId: 1,
    language: "cpp",
    code: "...",
  });
  console.log("提交成功", submission.id);
} catch (error) {
  console.error("提交失败", error.message);
}
```

### 3. 错误处理

```typescript
// ✅ 推荐：只捕获业务错误
try {
  const data = await apiGet<T>(url, true);
  // 处理成功逻辑
} catch (error) {
  // error.message 已经是友好的错误提示
  // 401/403/500 已自动处理，这里只需要显示 message
  showToast(error.message);
}

// ❌ 不推荐：过度处理
try {
  const data = await apiGet<T>(url, true);
} catch (error) {
  if (error.status === 401) {
    // 不需要，已自动跳转登录
  }
}
```
