### 1. 后端代码审计

#### 1.1 LocalJudgeService (不安全的本地判题)

**文件**: `backend/src/main/java/com/qoj/module/judge/service/LocalJudgeService.java`

**状态**: ✅ 已标记 @Deprecated

**安全警告**:
```java
/**
 * ⚠️⚠️⚠️ 严重安全警告 ⚠️⚠️⚠️
 * 本服务在主服务器上直接执行用户提交的代码，存在严重的安全风险
 * ⚠️ 生产环境绝对禁止启用此服务 ⚠️
 */
@Service
@ConditionalOnProperty(name = "qoj.judge.enable-unsafe-local-judge", havingValue = "true")
@Deprecated(since = "0.2.0", forRemoval = true)
public class LocalJudgeService {
```

**使用情况**:
- `SubmissionService` 中有条件引用（通过 @ConditionalOnProperty 控制）
- 仅在配置 `qoj.judge.enable-unsafe-local-judge=true` 时启用

**建议**: ✅ **保留但禁用**
- 已正确标记为不安全且废弃
- 已通过 @ConditionalOnProperty 默认禁用
- 保留用于本地开发测试
- 生产环境必须使用 DOMjudge 或 Docker 判题

#### 1.2 SubmissionCase / SubmissionCaseResult

**文件**:
- `backend/src/main/java/com/qoj/module/submission/entity/SubmissionCase.java`
- `backend/src/main/java/com/qoj/module/submission/entity/SubmissionCaseResult.java`

**状态**: ✅ **正在使用中（非无用代码）**

**使用情况**:

**SubmissionCase** 使用者:
1. `LocalJudgeService` - 本地判题写入测试点结果
2. `ContestService` - 比赛榜单计算读取测试点结果
3. `SubmissionService` - 查询提交详情时返回测试点信息

**SubmissionCaseResult** 使用者:
1. `JudgeCallbackService` - 判题回调时保存详细测试用例结果

**数据模型差异**:
```
SubmissionCase (简化版):
- 仅存储基本信息: caseNo, status, timeMs, memoryKb
- 用于 LocalJudgeService 本地判题

SubmissionCaseResult (完整版):
- 包含详细信息: subtaskNo, score, maxScore, inputPreview, outputPreview, judgeMessage
- 用于外部判题系统（DOMjudge）回调
```

**建议**: ✅ **保留两个实体**
- `SubmissionCase`: 用于 LocalJudgeService（虽然已废弃，但本地测试仍需）
- `SubmissionCaseResult`: 用于生产环境的外部判题回调

- 用于首次加载时的默认状态

**代码分析**:
```typescript
// OjDataProvider.tsx
function getInitialState(): OjState {
  const stored = window.localStorage.getItem(STORAGE_KEY);
  if (!stored) {
    return { ...defaultOjState, ratings: [] }; // 使用 mockData
  }
  return { ...defaultOjState, ...JSON.parse(stored), ratings: [] };
}

useEffect(() => {
  hydrateStateFromApi(state).then((next) => {
    setState(next); // 用真实数据覆盖
```
