# QOJ 数据库文档

QOJ 使用 MySQL 8.0 数据库，通过 Flyway 管理数据库版本演进。本文档详细说明表结构、关系、索引及迁移历史。

---

## 数据库配置

**数据库引擎**: MySQL 8.0  
**字符集**: UTF8MB4  
**排序规则**: utf8mb4_unicode_ci  
**时区**: UTC  

**Docker 启动**:
```bash
docker compose -f .runtime/qoj-deps.compose.yml up -d mysql
```

**连接参数**:
- Host: `127.0.0.1`
- Port: `13306`
- Database: `qoj`
- Username: `root`
- Password: `root`

## 核心表结构

### 1. users（用户表）

```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(80) NOT NULL UNIQUE COMMENT '用户名（登录账号）',
    student_no VARCHAR(80) UNIQUE COMMENT '学号',
    email VARCHAR(160) UNIQUE COMMENT '邮箱',
    password_hash VARCHAR(120) NOT NULL COMMENT 'BCrypt 密码哈希',
    role VARCHAR(32) NOT NULL COMMENT 'STUDENT/TEACHER/CLUB_ADMIN/SUPER_ADMIN',
    display_name VARCHAR(80) NOT NULL COMMENT '显示名称',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) COMMENT '用户表';
```

**索引**:
- PRIMARY KEY: `id`
- UNIQUE KEY: `username`, `student_no`, `email`

**关系**:
- → `problems.owner_id`: 用户创建的题目
- → `contests.owner_id`: 用户创建的比赛
- → `submissions.user_id`: 用户的提交记录

---

### 2. problems（题库表）

```sql
CREATE TABLE problems (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL COMMENT '题目标题',
    statement LONGTEXT NOT NULL COMMENT '题面（支持 Markdown + KaTeX）',
    input_format TEXT COMMENT '输入格式说明',
    output_format TEXT COMMENT '输出格式说明',
    sample_cases JSON COMMENT '样例数据',
    time_limit INT NOT NULL COMMENT '时间限制（毫秒）',
    memory_limit INT NOT NULL COMMENT '内存限制（MB）',
    difficulty TINYINT NOT NULL COMMENT '难度（1-5）',
    tags JSON COMMENT '标签数组',
    owner_id BIGINT NOT NULL COMMENT '创建者 ID',
    is_public BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否公开',
    is_deleted BOOLEAN NOT NULL DEFAULT FALSE COMMENT '软删除标记',
    domjudge_problem_id VARCHAR(50) COMMENT 'DOMjudge 题目 ID',
    ac_rate DECIMAL(5,2) NOT NULL DEFAULT 0 COMMENT 'AC 通过率',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    CONSTRAINT fk_problems_owner FOREIGN KEY (owner_id) REFERENCES users(id),
    FULLTEXT KEY ft_problems_title_statement (title, statement)
) COMMENT '题库表';
```

**索引**:
- PRIMARY KEY: `id`
- FOREIGN KEY: `owner_id` → `users.id`
- INDEX: `idx_problems_deleted_public_difficulty (is_deleted, is_public, difficulty)`
- FULLTEXT: `ft_problems_title_statement (title, statement)`

---

### 3. contests（比赛表）

```sql
CREATE TABLE contests (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    start_time DATETIME NOT NULL COMMENT '开始时间',
    end_time DATETIME NOT NULL COMMENT '结束时间',
    freeze_time DATETIME NULL COMMENT '封榜时间（NULL 表示不封榜）',
    type VARCHAR(16) NOT NULL COMMENT '赛制（ACM/OI）',
    scoring_mode VARCHAR(16) NOT NULL DEFAULT 'ACM' COMMENT 'ACM or OI',
    owner_id BIGINT NOT NULL,
    audience VARCHAR(16) NOT NULL COMMENT '受众类型（ALL/CLASS/CLUB）',
    audience_id BIGINT COMMENT '受众 ID（班级/社团）',
    frozen BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否封榜',
    allow_fullscreen BOOLEAN NOT NULL DEFAULT FALSE,
    anti_cheat_enabled BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否启用防切屏',
    max_switches INT NOT NULL DEFAULT 3 COMMENT '允许切屏次数上限',
    registration_type VARCHAR(32) NOT NULL DEFAULT 'PUBLIC',
    registration_password VARCHAR(255) NULL COMMENT '报名密码（加密）',
    status VARCHAR(16) NOT NULL DEFAULT 'NOT_STARTED',
    penalty_minutes INT NOT NULL DEFAULT 20 COMMENT 'ACM 罚时（分钟）',
    count_ce_as_penalty BOOLEAN NOT NULL DEFAULT FALSE COMMENT 'CE 是否计入罚时',
    scoreboard_scope VARCHAR(32) NOT NULL DEFAULT 'PUBLIC',
    scoreboard_visibility VARCHAR(32) NOT NULL DEFAULT 'PUBLIC',
    code_visibility VARCHAR(32) NOT NULL DEFAULT 'AFTER_CONTEST',
    enable_rolling_scoreboard BOOLEAN NOT NULL DEFAULT FALSE,
    is_deleted BOOLEAN NOT NULL DEFAULT FALSE,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    CONSTRAINT fk_contests_owner FOREIGN KEY (owner_id) REFERENCES users(id)
) COMMENT '比赛表';
```

**索引**:
- PRIMARY KEY: `id`
- FOREIGN KEY: `owner_id` → `users.id`
- INDEX: `idx_contests_deleted_status_time (is_deleted, status, start_time DESC)`
- INDEX: `idx_contests_time_range (start_time, end_time)`

### 4. submissions（代码提交表）

```sql
CREATE TABLE submissions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    problem_id BIGINT NOT NULL,
    contest_id BIGINT COMMENT '比赛 ID（NULL 表示普通提交）',
    practice_id BIGINT COMMENT '练习集 ID',
    participant_id BIGINT COMMENT '参赛者 ID',
    code MEDIUMTEXT NOT NULL COMMENT '提交代码',
    code_length INT COMMENT '代码长度',
    language VARCHAR(40) NOT NULL COMMENT '编程语言',
    status VARCHAR(40) NOT NULL COMMENT '判题状态（PENDING/AC/WA/TLE/CE/RE/MLE）',
    score INT NULL COMMENT '得分（OI/部分分）',
    time_used INT COMMENT '运行时间（毫秒）',
    memory_used INT COMMENT '内存使用（KB）',
    identity_type VARCHAR(16) COMMENT '提交身份类型',
    identity_id BIGINT COMMENT '提交身份 ID',
    domjudge_submission_id VARCHAR(80) COMMENT 'DOMjudge 提交 ID',
    submit_time DATETIME COMMENT '提交时间（冗余字段）',
    judge_start_time DATETIME COMMENT '判题开始时间',
    judge_end_time DATETIME COMMENT '判题结束时间',
    is_contest_submission BOOLEAN NOT NULL DEFAULT FALSE,
    is_frozen BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否在封榜期间提交',
    is_rejudged BOOLEAN NOT NULL DEFAULT FALSE,
    judge_message TEXT COMMENT '判题信息',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    CONSTRAINT fk_submissions_user FOREIGN KEY (user_id) REFERENCES users(id),
    CONSTRAINT fk_submissions_problem FOREIGN KEY (problem_id) REFERENCES problems(id),
    CONSTRAINT fk_submissions_contest FOREIGN KEY (contest_id) REFERENCES contests(id),
    CONSTRAINT fk_submissions_practice FOREIGN KEY (practice_id) REFERENCES practices(id),
    CONSTRAINT fk_submissions_participant FOREIGN KEY (participant_id) 
        REFERENCES contest_participants(id) ON DELETE SET NULL
) COMMENT '代码提交表（包括普通提交、比赛提交、练习提交）';
```

**索引**:
- PRIMARY KEY: `id`
- INDEX: `idx_submissions_user_problem_status (user_id, problem_id, status)`
- INDEX: `idx_submissions_contest_user (contest_id, user_id)`
- INDEX: `idx_submissions_domjudge_submission_id (domjudge_submission_id)`
- INDEX: `idx_user_status_time (user_id, status, created_at DESC)`
- INDEX: `idx_problem_status (problem_id, status)`
- INDEX: `idx_contest_submit_time (contest_id, submit_time)`
- INDEX: `idx_participant (participant_id)`
- INDEX: `idx_frozen (contest_id, is_frozen)`

### 5. contest_acm_rank_cache（ACM 榜单缓存）

```sql
CREATE TABLE contest_acm_rank_cache (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    contest_id BIGINT NOT NULL COMMENT '比赛 ID',
    participant_id BIGINT NOT NULL COMMENT '参赛者 ID',
    rank_no INT NOT NULL DEFAULT 0 COMMENT '排名',
    solved_count INT NOT NULL DEFAULT 0 COMMENT '通过题数',
    penalty_time INT NOT NULL DEFAULT 0 COMMENT '罚时（分钟）',
    submission_count INT NOT NULL DEFAULT 0 COMMENT '总提交次数',
    last_ac_time DATETIME NULL COMMENT '最后 AC 时间',
    last_submit_time DATETIME NULL COMMENT '最后提交时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_contest_participant (contest_id, participant_id),
    KEY idx_contest (contest_id),
    KEY idx_rank (contest_id, solved_count DESC, penalty_time ASC, last_ac_time ASC)
) COMMENT 'ACM 比赛榜单缓存（实时更新）';
```

**关联表**:
- `contest_acm_rank_problems`: 单题状态（是否 AC、错误次数、AC 时间）

### 6. contest_oi_rank_cache（OI 榜单缓存）

```sql
CREATE TABLE contest_oi_rank_cache (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    contest_id BIGINT NOT NULL,
    participant_id BIGINT NOT NULL,
    rank_no INT NOT NULL DEFAULT 0,
    total_score INT NOT NULL DEFAULT 0 COMMENT '总分',
    solved_count INT NOT NULL DEFAULT 0 COMMENT '满分题数',
    submission_count INT NOT NULL DEFAULT 0,
    last_score_update_time DATETIME NULL COMMENT '最后得分更新时间',
    last_submit_time DATETIME NULL,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_contest_participant (contest_id, participant_id),
    KEY idx_contest (contest_id),
    KEY idx_rank (contest_id, total_score DESC, solved_count DESC, last_score_update_time ASC)
) COMMENT 'OI 比赛榜单缓存（实时更新）';
```

**关联表**:
- `contest_oi_rank_problems`: 单题最高分（最高分、最高分提交 ID、首次满分时间）

### 关键迁移详解

#### V15: ACM/OI 榜单系统

**新增表**:
- `contest_participants`: 参赛者表（支持多身份参赛）
- `contest_acm_rank_cache`: ACM 榜单缓存
- `contest_acm_rank_problems`: ACM 单题状态
- `contest_oi_rank_cache`: OI 榜单缓存
- `contest_oi_rank_problems`: OI 单题分数
- `contest_scoreboard_snapshots`: 榜单快照（封榜、最终榜）

**数据迁移**:
```sql
-- 从旧的 contest_registrations 迁移数据到 contest_participants
INSERT INTO contest_participants (contest_id, user_id, identity_type, identity_id, nickname)
SELECT cr.contest_id, cr.user_id, cr.identity_type, cr.identity_id, u.display_name
FROM contest_registrations cr
INNER JOIN users u ON cr.user_id = u.id;
```

**触发器**:
```sql
CREATE TRIGGER before_submission_insert
BEFORE INSERT ON submissions
FOR EACH ROW
BEGIN
    IF NEW.contest_id IS NOT NULL THEN
        SET NEW.is_contest_submission = TRUE;
        -- 自动关联 participant_id
        SELECT id INTO NEW.participant_id
        FROM contest_participants
        WHERE contest_id = NEW.contest_id AND user_id = NEW.user_id;
    END IF;
END;
```

#### V20: 索引优化

**添加的复合索引**:
```sql
-- 用户提交历史查询优化
CREATE INDEX idx_user_status_time ON submissions(user_id, status, created_at DESC);

-- 题目 AC 率统计优化
CREATE INDEX idx_problem_status ON submissions(problem_id, status);

-- 软删除 + 公开状态复合索引
CREATE INDEX idx_problems_deleted_public_difficulty ON problems(is_deleted, is_public, difficulty);

-- 比赛时间范围查询
CREATE INDEX idx_contests_time_range ON contests(start_time, end_time);
```

**外键约束**:
```sql
ALTER TABLE submissions 
ADD CONSTRAINT fk_submissions_participant
FOREIGN KEY (participant_id) REFERENCES contest_participants(id) ON DELETE SET NULL;
```

---

#### V21: 统一测试点表

**背景**: 早期存在两张测试点表：
- `submission_cases`: 简化版（仅 status, time_ms, memory_kb）
- `submission_case_results`: 完整版（含 score, subtask_no, judge_message）

**迁移策略**:
```sql
-- 1. 数据迁移
INSERT INTO submission_case_results (submission_id, case_no, status, time_ms, memory_kb)
SELECT submission_id, case_no, status, COALESCE(time_ms, 0), COALESCE(memory_kb, 0)
FROM submission_cases
WHERE NOT EXISTS (
    SELECT 1 FROM submission_case_results scr
    WHERE scr.submission_id = submission_cases.submission_id
      AND scr.case_no = submission_cases.case_no
);

-- 2. 删除旧表
DROP TABLE IF EXISTS submission_cases;
```

## 软删除机制

QOJ 对以下资源实现软删除（`is_deleted` 标记）：
- `problems`: 题目软删除（V16）
- `practices`: 练习集软删除（V17）
- `contests`: 比赛软删除（V18）

**查询时自动过滤**:
```java
// 示例：ProblemService.java
LambdaQueryWrapper<Problem> query = new LambdaQueryWrapper<>();
query.eq(Problem::getIsDeleted, false);  // 只查询未删除的题目
```

**恢复软删除**:
```sql
UPDATE problems SET is_deleted = false WHERE id = 123;
```

## 索引优化建议

### 查询性能优化

**1. 用户提交历史**:
```sql
-- 已优化：idx_user_status_time (user_id, status, created_at DESC)
SELECT * FROM submissions 
WHERE user_id = ? AND status = 'AC' 
ORDER BY created_at DESC 
LIMIT 20;
```

**2. 题目 AC 率统计**:
```sql
-- 已优化：idx_problem_status (problem_id, status)
SELECT COUNT(*) FROM submissions 
WHERE problem_id = ? AND status = 'AC';
```

**3. 比赛榜单排序**:
```sql
-- 已优化：idx_rank (contest_id, solved_count DESC, penalty_time ASC)
SELECT * FROM contest_acm_rank_cache 
WHERE contest_id = ? 
ORDER BY solved_count DESC, penalty_time ASC;
```

## 数据备份与恢复

### 备份数据库

```bash
# 导出所有表结构和数据
docker exec qoj-mysql mysqldump -uroot -proot qoj > qoj_backup_$(date +%Y%m%d).sql

# 仅导出表结构
docker exec qoj-mysql mysqldump -uroot -proot --no-data qoj > qoj_schema.sql
```

```bash
# 从备份恢复
docker exec -i qoj-mysql mysql -uroot -proot qoj < qoj_backup_20240613.sql

# 重置数据库（危险操作！）
docker exec -i qoj-mysql mysql -uroot -proot -e "DROP DATABASE IF EXISTS qoj; CREATE DATABASE qoj CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

## 常见问题

### 1. Flyway 迁移失败：Checksum mismatch

**原因**: 已执行的迁移脚本被修改。

**解决**:
```sql
-- 查看 Flyway 历史
SELECT * FROM flyway_schema_history ORDER BY installed_rank;

-- 删除失败的迁移记录（谨慎操作）
DELETE FROM flyway_schema_history WHERE version = '21';
```

### 2. 外键约束错误

**原因**: 数据不一致，存在孤儿记录。

**解决**:
```sql
-- 检查孤儿记录
SELECT * FROM submissions s
LEFT JOIN contest_participants cp ON s.participant_id = cp.id
WHERE s.participant_id IS NOT NULL AND cp.id IS NULL;

-- 清理孤儿记录
UPDATE submissions SET participant_id = NULL 
WHERE participant_id NOT IN (SELECT id FROM contest_participants);
```

### 3. 索引未生效

**原因**: 统计信息过期。

**解决**:
```sql
-- 更新表统计信息
ANALYZE TABLE submissions;
ANALYZE TABLE problems;
ANALYZE TABLE contests;
```
