## 发现的安全问题

### 1. **中等漏洞：管理接口权限混乱**

**问题描述**:
- `ContestController` (用户接口) 中包含 `create` 和 `update` 方法
- 使用 `@PreAuthorize` 注解进行权限控制
- 管理操作混在用户接口中，不符合 RESTful 规范

**代码位置**:
```java
// ContestController.java - /api/v1/contests
@PostMapping
@PreAuthorize("hasAnyRole('TEACHER','CLUB_ADMIN','SUPER_ADMIN')")
public ApiResponse<ContestVO> create(@Valid @RequestBody ContestCreateRequest request) {
    return ApiResponse.ok(contestService.create(request));
}

@PutMapping("/{id}")
@PreAuthorize("hasAnyRole('TEACHER','CLUB_ADMIN','SUPER_ADMIN')")
public ApiResponse<ContestVO> update(@PathVariable long id, @Valid @RequestBody ContestUpdateRequest request) {
    return ApiResponse.ok(contestService.update(id, request));
}
```

**影响**:
- API 路径混乱（管理操作在用户路径下）
- 前端需要区分权限才能显示按钮
- 不符合项目统一规范（管理接口应在 `/api/admin/v1/*`）

### 2. **严重漏洞：比赛提交验证不完整** ⚠️

**问题描述**:
- `ContestService.requireSubmittableContestProblem()` 只检查了 4 项
- **缺少时间范围检查**：未验证 `startTime <= now <= endTime`
- **缺少状态检查**：未验证 `status = RUNNING`
- 用户可能在比赛未开始或已结束时提交

**现有验证**:
```java
public ContestProblem requireSubmittableContestProblem(Long contestId, Long submittedProblemId, Long userId) {
    // ✅ 1. 比赛存在
    Contest contest = contestMapper.selectById(contestId);
    
    // ✅ 2. 题目属于比赛
    ContestProblem contestProblem = resolveContestProblem(contestId, submittedProblemId);
    
    // ✅ 3. 用户已报名
    ContestRegistration registration = registrationForUser(contestId, userId);
    
    // ✅ 4. 报名身份有效
    resolveRegistrationIdentity(contest, type, registration.identityId, userId);
    
    // ❌ 缺少时间范围检查
    // ❌ 缺少状态检查
}
```

**攻击场景**:
```bash
POST /api/v1/submissions
{
  "contestId": 1,        # 比赛已结束
  "problemId": 101,
  "code": "...",
  "language": "cpp"
}
# 如果比赛未检查时间和状态，提交可能成功
```

### 3. **功能缺失：软删除未实现**

**问题描述**:
- Contest 表没有 `is_deleted` 和 `deleted_at` 字段
- 删除操作是物理删除（永久丢失数据）
- 无法恢复误删除的比赛
- 已有提交记录的比赛删除会引发外键约束错误

### 4. **轻度问题：比赛题目时间限制已实现**

**检查结果**:
- ✅ `ContestAccessPolicy.canViewProblemDetail()` 已实现时间检查
- ✅ `ContestService.problemDetail()` 正确调用 Policy
- ✅ 比赛未开始前，普通用户无法查看题目详情

```java
public ContestProblem requireSubmittableContestProblem(Long contestId, Long submittedProblemId, Long userId) {
    // 1. 验证比赛存在
    Contest contest = contestMapper.selectById(contestId);
    if (contest == null) {
        throw new BizException(ErrorCode.NOT_FOUND, "比赛不存在");
    }

    // 2. ✅ 验证当前时间在比赛时间范围内
    LocalDateTime now = LocalDateTime.now();
    if (now.isBefore(contest.startTime)) {
        throw new BizException(ErrorCode.FORBIDDEN, "比赛尚未开始");
    }
    if (now.isAfter(contest.endTime)) {
        throw new BizException(ErrorCode.FORBIDDEN, "比赛已结束");
    }

    // 3. ✅ 验证比赛状态为 RUNNING
    if (!ContestStatus.RUNNING.name().equals(contest.status)) {
        throw new BizException(ErrorCode.FORBIDDEN, "比赛未在进行中");
    }

    // 4. 验证题目属于该比赛
    ContestProblem contestProblem = resolveContestProblem(contestId, submittedProblemId);
    if (contestProblem == null) {
        throw new BizException(ErrorCode.BAD_REQUEST, "该题不属于当前比赛");
    }

    // 5. 验证用户已报名
    ContestRegistration registration = registrationForUser(contestId, userId);
    if (registration == null) {
        throw new BizException(ErrorCode.FORBIDDEN, "请先报名比赛");
    }

    // 6. 验证报名身份有效
    IdentityType type;
    try {
        type = IdentityType.valueOf(registration.identityType);
    } catch (Exception ex) {
        throw new BizException(ErrorCode.BAD_REQUEST, "报名身份数据异常，请重新报名");
    }
    resolveRegistrationIdentity(contest, type, registration.identityId, userId);

    return contestProblem;
}
```

**验证逻辑完整度**:
1. ✅ 比赛存在性检查
2. ✅ **时间范围检查**（新增）
3. ✅ **状态检查**（新增）
4. ✅ 题目归属检查
5. ✅ 用户报名检查
6. ✅ 报名身份验证

**缺失项**（后续可添加）:

public ContestVO detail(long id) {
    Contest contest = contestMapper.selectById(id);
    if (contest == null) {
        throw new BizException(ErrorCode.NOT_FOUND, "比赛不存在");
    }

    // ✅ 检查软删除（只有超级管理员和创建者能看到已删除的比赛）
    if (Boolean.TRUE.equals(contest.isDeleted)) {
        AuthUser user = CurrentUser.get();
        if (user == null) {
            throw new BizException(ErrorCode.NOT_FOUND, "比赛不存在");
        }
        boolean isOwner = contest.ownerId.equals(user.id())
            && ((user.adminAccount() && "ADMIN".equals(contest.ownerAccountType))
                || (!user.adminAccount() && "USER".equals(contest.ownerAccountType)));
        if (!"SUPER_ADMIN".equals(user.role()) && !isOwner) {
            throw new BizException(ErrorCode.NOT_FOUND, "比赛不存在");
        }
    }

    return toVO(contest);
}
```

 * 软删除比赛

@Transactional
public void delete(long id) {
    Contest contest = contestMapper.selectById(id);
    if (contest == null) {
        throw new BizException(ErrorCode.NOT_FOUND, "比赛不存在");
    }

    // 软删除
    contest.isDeleted = true;
    contest.deletedAt = LocalDateTime.now();
    contestMapper.updateById(contest);
}
```

#### 3.5 添加删除接口

**修改文件**: `AdminContestController.java`

```java
@DeleteMapping("/{id}")
public ApiResponse<Void> delete(@PathVariable long id) {
    Contest contest = contestMapper.selectById(id);
    if (contest == null) {
        throw new BizException(ErrorCode.NOT_FOUND, "比赛不存在");
    }

    AuthUser user = CurrentUser.required();
    if (!contestAccessPolicy.can(user, Permission.DELETE, contest)) {
        throw new BizException(ErrorCode.FORBIDDEN, "无权删除该比赛");
    }

    contestService.delete(id);
    return ApiResponse.ok();
}
```
