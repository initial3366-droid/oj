### 阶段4: 修复安全漏洞

**已验证的安全机制**:

#### 4.1 私有题目防泄露
- 通过 `ProblemAccessPolicy` 控制访问
- 未公开题目只有创建者和超级管理员可见
- 已登录用户无法通过 ID 枚举访问私有题目

#### 4.2 比赛访问控制
- 比赛未开始：只有创建者和超级管理员可查看题目
- 比赛已结束：不允许提交
- 比赛进行中：必须已报名才能提交
- 题目归属校验：`requireSubmittableContestProblem` 验证题目属于比赛

**核心校验代码**:
```java
public ContestProblem requireSubmittableContestProblem(Long contestId, Long problemId, Long userId) {
    // 1. 验证比赛存在
    // 2. 验证时间范围（未开始/已结束）
    // 3. 验证比赛状态为 RUNNING
    // 4. 验证题目属于该比赛
    // 5. 验证用户已报名
    // 6. 验证报名身份有效
}
```

#### 4.3 练习提交校验
- 验证练习是否存在且已发布
- 验证题目属于该练习（`practiceProblemMapper.selectCount`）
- 验证用户有提交权限（`practiceAccessPolicy.can`）
- 验证 audience 权限（班级/社团成员）

#### 4.4 ACM/OI 榜单统一
- V15 引入统一榜单：`contest_acm_rank_cache`、`contest_oi_rank_cache`
- V19 废弃旧表：`contest_standings` → `_deprecated_contest_standings`
- V22 删除废弃表
- 代码中无旧榜单引用

**风险**: 无

### 阶段5: 重构判题模块

**架构设计**:
```
JudgeService (接口)
    ├── DomjudgeAdapter (推荐) - 远程 DOMjudge 判题
    ├── DockerJudgeService - Docker 容器隔离判题
    └── LocalJudgeService (不安全，已废弃) - 本地直接执行
```

**修改文件**:
- `QojProperties.java` - 更新 `Judge` 配置类

**新增配置**:
```yaml
qoj:
  judge:
    # 判题模式（推荐 domjudge）
    mode: domjudge  # domjudge | docker
    
    # ⚠️ 不安全的本地判题开关（仅开发环境）
    enable-unsafe-local-judge: false
    
    # 沙箱调试开关
    enable-sandbox: false
```

**安全保护**:
- `LocalJudgeService` 已标记 `@Deprecated(since = "0.2.0", forRemoval = true)`
- 使用 `@ConditionalOnProperty(name = "qoj.judge.enable-unsafe-local-judge", havingValue = "true")`
- 默认禁用，生产环境严禁启用

**风险**: 无

## 四、系统架构改进

### 4.1 权限控制三层模型

**第一层：URL 层** (SecurityConfig)
```java
http.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/admin/**").hasAnyRole("SUPER_ADMIN", "TEACHER", "CLUB_ADMIN")
    .requestMatchers("/api/v1/**").permitAll()
)
```

**第二层：角色层** (Controller @PreAuthorize)
```java
@PreAuthorize("hasRole('SUPER_ADMIN')")
public ApiResponse<UserVO> deleteUser(@PathVariable long id) { ... }
```

**第三层：资源层** (AccessPolicy)
```java
// ProblemAccessPolicy.canView
if (Boolean.TRUE.equals(problem.isPublic)) return true;
if (problem.ownerId != null && problem.ownerId.equals(user.id())) return true;
return false; // 私有题目且非创建者
```

### 4.2 判题架构

```
┌─────────────────┐
│ SubmissionService│
└────────┬────────┘
         │
         ├─→ DomjudgeAdapter (推荐)
         │      └─→ 远程 DOMjudge API
         │
         ├─→ DockerJudgeService
         │      └─→ Docker 容器隔离
         │
         └─→ LocalJudgeService (已废弃)
                └─→ 本地直接执行 ⚠️
```

**当前表结构**: 30+ 张表，包括：
