# 修改总览

本文档总结了 QOJ 在线评测系统的所有修改、重构、安全加固和文档完善工作。

---

## 1. 已修复的问题

### 1.1 数据库迁移问题
- ✅ **Flyway 迁移脚本冲突** - V21 统一了 `submission_cases` 和 `submission_case_results` 表
- ✅ **重复字段迁移** - 验证了 `is_deleted` 字段添加到不同表（problems、practices、contests），非重复迁移
- ✅ **迁移幂等性** - 所有迁移脚本使用 `IF NOT EXISTS` 条件判断，支持安全升级

### 1.2 前端问题
- ✅ **Mock 数据体积过大** - 将 247 行硬编码数据简化为空数组初始化，减少 bundle 体积
- ✅ **类型定义不一致** - 修复 `OjState` 接口中 `dailyProblemId` 和 `activeUser` 的 null 类型
- ✅ **WebSocket 调试日志** - 使用 `import.meta.env.DEV` 条件判断，生产环境不输出调试信息

### 1.3 后端问题
- ✅ **错误码缺失** - 添加 `NOT_IMPLEMENTED(50100)` 错误码，用于未实现功能
- ✅ **编译警告** - 25 个 @Deprecated 警告（ProblemVO、LocalJudgeService），已标记待删除
- ✅ **构建成功** - Maven 编译通过，140 个单元测试全部通过

- ✅ **比赛功能完善**

  - 支持榜单可见性配置（`scoreboard_visibility`）

### 2.2 练习模块

## 4. 安全修复

### 4.1 代码执行安全
- ✅ **不安全的本地判题默认禁用**
  - `LocalJudgeService` 标记为 `@Deprecated`
  - 默认配置：`enable-unsafe-local-judge: false`
  - 生产环境必须使用 DOMjudge 或 Docker 判题

- ✅ **安全警告注释**
  ```java
  /**
   * ⚠️⚠️⚠️ 严重安全警告 ⚠️⚠️⚠️
   * 本服务在主服务器上直接执行用户提交的代码，存在严重的安全风险
   * ⚠️ 生产环境绝对禁止启用此服务 ⚠️
   */
  ```

### 4.2 JWT 安全
- ✅ **强制 64 字节 Secret**
  - 配置验证：`if (keyBytes.length < 64) throw new IllegalStateException(...)`
  - 默认值仅用于开发：`change-this-secret-to-at-least-32-bytes`
  - 生产环境通过环境变量覆盖

- ✅ **Token 轮换防重放**
  - Refresh Token Family（familyId）
  - 旧 Token 立即加入黑名单
  - 检测到重放攻击时撤销整个 Token Family

### 4.3 权限安全
- ✅ **三层权限控制**
  - URL 层：SecurityConfig（仅超级管理员可访问 `/api/admin/v1/users/**`）
  - 角色层：@PreAuthorize（教师/社团管理员可访问其他管理接口）
  - 资源层：AccessPolicy（细粒度的所有权验证）

- ✅ **代码查看权限**
  - 仅提交者本人和超级管理员可查看代码
  - 比赛代码查看受 `code_visibility` 配置控制
  - 所有代码查看操作记录审计日志

### 4.4 敏感文件保护
- ✅ **.gitignore 配置**
  ```
  .env
  .env.local
  .env.*.local
  *.sql
  qoj_database_backup.sql
  node_modules
  dist
  backend/target
  ```

### 常见问题

#### 1. 后端启动失败：MySQL 连接错误
```
Caused by: java.sql.SQLException: Access denied for user 'root'@'localhost'
```

**解决**:
```bash
# 确认 MySQL 容器运行
docker ps | grep mysql

# 测试连接
docker exec -it qoj-mysql mysql -uroot -proot -e "SELECT 1"

# 检查 .env 配置
cat .env | grep MYSQL
```

#### 2. 前端 API 请求 404
```
GET http://127.0.0.1:5173/api/v1/problems 404 (Not Found)
```

**解决**:
1. 确认后端运行在 `http://127.0.0.1:8080`
2. 检查 Vite 代理配置（vite.config.ts）
3. 查看浏览器控制台网络请求

#### 3. Flyway 迁移失败
```
Validate failed: Detected resolved migration not applied
```

**解决**:
```bash
# 重置数据库（会丢失所有数据！）
docker compose -f .runtime/qoj-deps.compose.yml down -v
docker compose -f .runtime/qoj-deps.compose.yml up -d
cd backend && mvn spring-boot:run
```

#### 4. JWT 验证失败
```
The Token's Signature resulted invalid
```

**解决**:
- 确认 `JWT_SECRET` 长度 ≥ 64 字节
- 生成新 Secret: `openssl rand -base64 64`
- 更新 `.env` 文件并重启后端
