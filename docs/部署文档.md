# QOJ 生产环境部署指南

本文档详细说明 QOJ 在线评测系统在生产环境的部署流程、配置要点、安全加固及运维监控。

---

## 部署架构

### 推荐架构（生产环境）

```
[用户] 
  ↓ HTTPS (443)
[Nginx 反向代理] 
  ↓ HTTP (8080)
[QOJ 后端 (Spring Boot)]
  ↓
[MySQL 8.0] [Redis 7] [DOMjudge 判题服务器]
```

**组件说明**:
- **Nginx**: 反向代理、SSL 终结、静态资源、负载均衡
- **QOJ 后端**: Spring Boot 应用，处理 API 请求
- **MySQL**: 数据持久化
- **Redis**: 缓存、Session、Token 黑名单
- **DOMjudge**: 远程判题服务器（沙箱隔离）

## 部署步骤

### 1. 环境准备

#### 1.1 安装依赖

**Ubuntu/Debian**:
```bash
# 更新软件源
sudo apt update && sudo apt upgrade -y

# 安装 Java 17
sudo apt install openjdk-17-jdk -y
java -version

# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install nodejs -y
node --version

# 安装 Nginx
sudo apt install nginx -y

# 安装 Docker & Docker Compose
curl -fsSL https://get.docker.com | sudo sh
sudo systemctl enable docker
sudo systemctl start docker
```

**CentOS/RHEL**:
```bash
sudo yum install java-17-openjdk nginx -y
```

#### 1.2 创建部署目录

```bash
sudo mkdir -p /opt/qoj
sudo chown $USER:$USER /opt/qoj
cd /opt/qoj
```

### 2. 数据库部署

#### 2.1 Docker 方式（推荐）

**创建 Docker Compose 配置**:
```bash
mkdir -p /opt/qoj/docker
cat > /opt/qoj/docker/docker-compose.yml << 'EOF'
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: qoj-mysql-prod
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: qoj
      MYSQL_USER: qoj
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    ports:
      - "127.0.0.1:3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./mysql.cnf:/etc/mysql/conf.d/custom.cnf
    command:
      - --character-set-server=utf8mb4
      - --collation-server=utf8mb4_unicode_ci
      - --max_connections=500
      - --innodb_buffer_pool_size=2G

  redis:
    image: redis:7-alpine
    container_name: qoj-redis-prod
    restart: always
    command: redis-server --requirepass ${REDIS_PASSWORD} --maxmemory 1gb --maxmemory-policy allkeys-lru
    ports:
      - "127.0.0.1:6379:6379"
    volumes:
      - redis_data:/data

volumes:
  mysql_data:
  redis_data:
EOF
```

**启动数据库**:
```bash
cd /opt/qoj/docker
export MYSQL_ROOT_PASSWORD=$(openssl rand -base64 32)
export MYSQL_PASSWORD=$(openssl rand -base64 32)
export REDIS_PASSWORD=$(openssl rand -base64 32)

# 保存密码到 .env 文件
cat > .env << EOF
MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
MYSQL_PASSWORD=${MYSQL_PASSWORD}
REDIS_PASSWORD=${REDIS_PASSWORD}
EOF

docker compose up -d
```

#### 2.2 MySQL 生产配置

```bash
cat > /opt/qoj/docker/mysql.cnf << 'EOF'
[mysqld]
# 字符集
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

# 性能优化
innodb_buffer_pool_size=2G
innodb_log_file_size=256M
max_connections=500
thread_cache_size=50

# 慢查询日志
slow_query_log=1
slow_query_log_file=/var/log/mysql/slow.log
long_query_time=2

# 二进制日志（备份恢复）
log_bin=/var/log/mysql/mysql-bin
expire_logs_days=7
EOF

docker restart qoj-mysql-prod
```

### 3. 后端部署

#### 3.1 克隆代码

```bash
cd /opt/qoj
git clone <repository-url> app
cd app
```

#### 3.2 配置生产环境变量

```bash
cat > /opt/qoj/app/.env.prod << 'EOF'
# MySQL 配置
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_DATABASE=qoj
MYSQL_USERNAME=qoj
MYSQL_PASSWORD=<从 docker/.env 获取>

# Redis 配置
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=<从 docker/.env 获取>

# JWT 配置（生产环境必须修改！）
JWT_SECRET=<使用 openssl rand -base64 64 生成>
JWT_ACCESS_EXPIRE=900
JWT_REFRESH_EXPIRE=604800

# DOMjudge 配置
DOMJUDGE_BASE_URL=http://domjudge.internal.example.com
DOMJUDGE_API_KEY=<DOMjudge API Key>
DOMJUDGE_DEFAULT_CONTEST_ID=1

# 判题模式（生产环境只能用 domjudge 或 docker）
JUDGE_MODE=domjudge
ENABLE_UNSAFE_LOCAL_JUDGE=false
ENABLE_SANDBOX=false

# CORS 配置
CORS_ALLOWED_ORIGINS=https://qoj.example.com

# Spring Boot 配置
SPRING_PROFILES_ACTIVE=prod
SERVER_PORT=8080
EOF
```

**生成强随机密钥**:
```bash
# JWT Secret（至少 64 字节）
openssl rand -base64 64

# MySQL/Redis 密码
openssl rand -base64 32
```

#### 3.3 构建后端

```bash
cd /opt/qoj/app/backend
mvn clean package -DskipTests

# 验证 JAR 文件
ls -lh target/qoj-backend-*.jar
```

#### 3.4 创建 Systemd 服务

```bash
sudo cat > /etc/systemd/system/qoj-backend.service << 'EOF'
[Unit]
Description=QOJ Backend Service
After=network.target mysql.service redis.service

[Service]
Type=simple
User=qoj
Group=qoj
WorkingDirectory=/opt/qoj/app/backend
EnvironmentFile=/opt/qoj/app/.env.prod
ExecStart=/usr/bin/java -Xms2g -Xmx4g -jar /opt/qoj/app/backend/target/qoj-backend-1.0.0.jar
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

# 安全限制
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/qoj/app/logs

[Install]
WantedBy=multi-user.target
EOF
```

**创建专用用户**:
```bash
sudo useradd -r -s /bin/false qoj
sudo chown -R qoj:qoj /opt/qoj/app
```

**启动服务**:
```bash
sudo systemctl daemon-reload
sudo systemctl enable qoj-backend
sudo systemctl start qoj-backend

# 查看日志
sudo journalctl -u qoj-backend -f
```

### 4. 前端部署

#### 4.1 构建前端

```bash
cd /opt/qoj/app

# 安装依赖
npm install

# 生产构建
npm run build

# 验证构建产物
ls -lh dist/
```

#### 4.2 配置 Nginx

```bash
sudo cat > /etc/nginx/sites-available/qoj << 'EOF'
upstream qoj_backend {
    server 127.0.0.1:8080 fail_timeout=10s max_fails=3;
}

server {
    listen 80;
    server_name qoj.example.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name qoj.example.com;

    # SSL 证书（Let's Encrypt）
    ssl_certificate /etc/letsencrypt/live/qoj.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/qoj.example.com/privkey.pem;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;

    # HSTS
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 前端静态文件
    root /opt/qoj/app/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript;
    gzip_min_length 1000;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://qoj_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://qoj_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_connect_timeout 7d;
        proxy_send_timeout 7d;
        proxy_read_timeout 7d;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "no-referrer-when-downgrade" always;

    # 访问日志
    access_log /var/log/nginx/qoj_access.log;
    error_log /var/log/nginx/qoj_error.log;
}
EOF

# 启用站点
sudo ln -s /etc/nginx/sites-available/qoj /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### 4.3 获取 SSL 证书（Let's Encrypt）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 获取证书
sudo certbot --nginx -d qoj.example.com

# 自动续期
sudo systemctl enable certbot.timer
```

### 5. DOMjudge 部署（判题服务器）

#### 5.1 独立服务器部署

在独立服务器上部署 DOMjudge：

```bash
# 使用 Docker 快速部署
docker run -d \
  --name domjudge \
  -p 80:80 \
  -e MYSQL_ROOT_PASSWORD=rootpw \
  -e MYSQL_PASSWORD=djpw \
  -e MYSQL_DATABASE=domjudge \
  -v domjudge-data:/var/lib/mysql \
  domjudge/domjudge:latest

# 访问 http://<domjudge-server-ip>/
# 默认管理员账号：admin / password
```

#### 5.2 获取 API Key

1. 登录 DOMjudge 管理后台
2. 进入 **Users** → **admin** → **API Key**
3. 复制 API Key
4. 更新 QOJ 后端 `.env.prod` 中的 `DOMJUDGE_API_KEY`

#### 5.3 配置 DOMjudge Contest

```bash
# 创建永久比赛用于普通提交
curl -X POST http://<domjudge-server>/api/v4/contests \
  -H "Authorization: Bearer <API_KEY>" \
  -d '{
    "name": "QOJ Default Contest",
    "shortname": "qoj-default",
    "start_time": "2024-01-01T00:00:00",
    "end_time": "2030-12-31T23:59:59"
  }'
```

## 安全加固

### 1. 防火墙配置

```bash
# UFW (Ubuntu)
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable

# 禁止直接访问后端
sudo ufw deny 8080/tcp

# 禁止外部访问数据库
sudo ufw deny 3306/tcp
sudo ufw deny 6379/tcp
```

### 2. 数据库安全

```bash
# 运行 MySQL 安全脚本
docker exec -it qoj-mysql-prod mysql_secure_installation

# 删除测试数据库和匿名用户
# 禁止 root 远程登录
# 设置强密码
```

### 3. JWT Secret 轮换

```bash
# 生成新 Secret
NEW_SECRET=$(openssl rand -base64 64)

# 更新 .env.prod
sed -i "s/JWT_SECRET=.*/JWT_SECRET=${NEW_SECRET}/" /opt/qoj/app/.env.prod

# 重启后端（会导致所有用户 Token 失效，需重新登录）
sudo systemctl restart qoj-backend
```

### 4. Redis 密码保护

确保 Redis 密码足够复杂：

```bash
# 修改 Redis 密码
docker exec -it qoj-redis-prod redis-cli
> CONFIG SET requirepass <new-strong-password>
> AUTH <new-strong-password>
> CONFIG REWRITE

# 更新后端配置
# 修改 .env.prod 中的 REDIS_PASSWORD
sudo systemctl restart qoj-backend
```

### 5. 定期更新依赖

```bash
# 更新系统包
sudo apt update && sudo apt upgrade -y

# 更新 Docker 镜像
cd /opt/qoj/docker
docker compose pull
docker compose up -d

# 更新前端依赖
cd /opt/qoj/app
npm audit fix
npm run build
```

## 备份与恢复

### 1. 数据库备份

**自动备份脚本**:
```bash
cat > /opt/qoj/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR=/opt/qoj/backups
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份 MySQL
docker exec qoj-mysql-prod mysqldump -uqoj -p<password> qoj | gzip > $BACKUP_DIR/mysql_$DATE.sql.gz

# 备份 Redis
docker exec qoj-redis-prod redis-cli -a <password> --rdb /data/dump.rdb SAVE
docker cp qoj-redis-prod:/data/dump.rdb $BACKUP_DIR/redis_$DATE.rdb

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.gz" -mtime +7 -delete
find $BACKUP_DIR -name "*.rdb" -mtime +7 -delete

echo "Backup completed: $DATE"
EOF

chmod +x /opt/qoj/backup.sh
```

**定时备份（Cron）**:
```bash
# 每天凌晨 2 点备份
crontab -e
0 2 * * * /opt/qoj/backup.sh >> /var/log/qoj_backup.log 2>&1
```

### 2. 恢复数据

**恢复 MySQL**:
```bash
gunzip < /opt/qoj/backups/mysql_20240613_020000.sql.gz | \
docker exec -i qoj-mysql-prod mysql -uqoj -p<password> qoj
```

**恢复 Redis**:
```bash
docker cp /opt/qoj/backups/redis_20240613_020000.rdb qoj-redis-prod:/data/dump.rdb
docker restart qoj-redis-prod
```

## 性能优化

### 1. JVM 参数调优

```bash
# 编辑 systemd 服务文件
sudo systemctl edit qoj-backend

# 添加 JVM 参数
[Service]
Environment="JAVA_OPTS=-Xms2g -Xmx4g -XX:+UseG1GC -XX:MaxGCPauseMillis=200"

sudo systemctl daemon-reload
sudo systemctl restart qoj-backend
```

### 2. MySQL 查询优化

```sql
-- 查看慢查询
SELECT * FROM mysql.slow_log ORDER BY start_time DESC LIMIT 10;

-- 分析查询计划
EXPLAIN SELECT * FROM submissions WHERE user_id = 1 AND status = 'AC';

-- 添加缺失的索引
CREATE INDEX idx_custom ON submissions(user_id, status, created_at);
```

### 3. Redis 缓存优化

```bash
# 设置最大内存和淘汰策略
docker exec qoj-redis-prod redis-cli -a <password> CONFIG SET maxmemory 2gb
docker exec qoj-redis-prod redis-cli -a <password> CONFIG SET maxmemory-policy allkeys-lru
```

## 常见问题

### 1. 服务无法启动

```bash
# 查看详细错误
sudo journalctl -u qoj-backend -n 100 --no-pager

# 检查端口占用
sudo lsof -i :8080

# 检查文件权限
ls -la /opt/qoj/app/backend/target/*.jar
```

### 2. 数据库连接失败

```bash
# 检查 MySQL 状态
docker ps | grep mysql

# 测试连接
docker exec -it qoj-mysql-prod mysql -uqoj -p<password> -e "SELECT 1"

# 检查防火墙
sudo ufw status
```

### 3. Nginx 502 Bad Gateway

```bash
# 检查后端是否运行
curl http://127.0.0.1:8080/api/v1/home/carousel

# 检查 SELinux（CentOS）
sudo setenforce 0

# 检查 Nginx 配置
sudo nginx -t
```
