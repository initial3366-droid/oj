## 🔴 修复的严重安全漏洞

### 1. 判题系统安全漏洞 (CVSS 9.8 - CRITICAL) 🔥

**问题**: LocalJudgeService 在主服务器直接执行用户代码，无任何隔离

**攻击向量**:
- 文件系统访问（可读写任意文件）
- 网络访问（可建立任意连接）
- 资源耗尽（fork/内存炸弹）
- 权限提升（利用系统漏洞）

**修复措施**:
- ✅ 实现 DockerJudgeService（7 层隔离防御）
- ✅ 标记 LocalJudgeService 为 @Deprecated
- ✅ 添加配置模式切换（domjudge/docker/unsafe-local）
- ✅ 生产环境强制禁用不安全判题
- ✅ 创建 qoj-judge Docker 镜像

**隔离机制**:
```bash
--network none        # 完全禁止网络
--memory 256m         # 强制内存限制
--cpus 1              # CPU 核心限制
--pids-limit 32       # 防止 fork 炸弹
-v path:/workspace:ro # 只读文件系统
```

**防御深度**: 0 层 → 7 层

### 2. Contest 提交验证漏洞 (HIGH)

**问题**: 用户可在比赛未开始或已结束时提交，缺少时间和状态检查

**修复措施**:
- ✅ 增强 requireSubmittableContestProblem() 到 6 层验证
  1. 比赛存在性
  2. 时间范围检查（now >= startTime && now <= endTime）
  3. 状态检查（status = RUNNING）
  4. 题目归属验证
  5. 用户报名验证
  6. 身份有效性验证

### 3. Practice 提交验证漏洞 (HIGH)

**问题**: 用户可提交任意题目到任意练习，无验证

**修复措施**:
- ✅ 实现 5 层验证
  1. 练习存在性
  2. 发布状态检查
  3. 提交权限验证
  4. Audience 成员验证
  5. 题目归属验证

## 🚀 部署指南

### 1. 生产环境配置

#### 1.1 判题系统配置

**推荐方式 1: DOMjudge（最安全）**
```yaml
# application-prod.yml
qoj:
  judge:
    mode: domjudge
    enable-unsafe-local-judge: false
    enable-sandbox: false
  
  domjudge:
    base-url: https://judge.example.com
    api-key: ${DOMJUDGE_API_KEY}
```

**推荐方式 2: Docker 判题**
```yaml
# application-prod.yml
qoj:
  judge:
    mode: docker
    enable-unsafe-local-judge: false
    enable-sandbox: false
    docker:
      image: qoj-judge:latest
```

#### 1.2 构建 Docker 镜像

```bash
# 构建判题镜像
cd docker/judge
docker build -t qoj-judge:latest .

# 测试镜像
docker run --rm qoj-judge:latest g++ --version
```

### 2. 数据库迁移

```bash
# 1. 启动数据库
docker compose -f .runtime/qoj-deps.compose.yml up -d

# 2. 执行迁移（包含 V20-V22）
cd backend
mvn flyway:migrate

# 3. 验证迁移状态
mvn flyway:info

# 4. 检查表结构
mysql -h 127.0.0.1 -P 13306 -u root -proot qoj \
  -e "SHOW TABLES LIKE '%deprecated%';"  # 应该为空
```

### 3. 后端启动

```bash
# 设置环境变量
export JUDGE_MODE=docker
export SPRING_PROFILES_ACTIVE=prod

# 启动后端
cd backend
mvn clean package -DskipTests
java -jar target/qoj-backend-0.1.0.jar
```

### 4. 前端启动

```bash
cd frontend
npm install
npm run build
npm run preview  # 或部署到 Nginx
```
