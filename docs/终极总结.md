## 🎯 核心成果

### ACM/OI 排名系统
完整实现了双赛制排名系统：

**ACM 模式**:
- 排序规则: 解题数↓ → 罚时↑ → 最后AC时间↑
- 罚时计算: AC时间 + 错误次数 × 罚时系数
- 幂等性: 已 AC 题目不再处理
- 事务性: 判题回调保证原子性

**OI 模式**:
- 排序规则: 总分↓ → 满分题数↓ → 最后得分时间↑
- 计分规则: 每题取历史最高分
- 幂等性: 只在分数提高时更新
- 增量计算: 使用分数差值更新总分

**通用功能**:
- ✅ 可从 submissions 表全量重建
- ✅ 封榜快照系统（freeze/final）
- ✅ 实时排名更新
- ✅ WebSocket 推送支持

### REST API 端点
| 端点 | 方法 | 权限 | 功能 |
|------|------|------|------|
| /api/contests/{id}/rank/acm | GET | 公开 | 获取 ACM 榜单 |
| /api/contests/{id}/rank/oi | GET | 公开 | 获取 OI 榜单 |
| /api/contests/{id}/rank/acm/rebuild | POST | 管理员 | 重建 ACM 榜单 |
| /api/contests/{id}/rank/oi/rebuild | POST | 管理员 | 重建 OI 榜单 |
| /api/contests/{id}/scoreboard/snapshot | POST | 管理员 | 创建快照 |
| /api/contests/{id}/scoreboard/snapshot/{type} | GET | 公开 | 获取快照 |
| /api/contests/{id}/scoreboard/snapshot/{type} | DELETE | 管理员 | 删除快照 |
