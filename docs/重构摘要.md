**问题**: 
- V20260614__contest_rank_system.sql 版本号跳跃过大（应为 V15）
- V15 与 V13 存在字段冲突（重复添加 contest_problems.id、主键等）

**修复**: 
- 重命名迁移文件为 `V15__contest_rank_system.sql`

**关键改动**:
```sql
SET @sql = IF(
    NOT EXISTS (
        SELECT 1 FROM information_schema.COLUMNS
        WHERE TABLE_SCHEMA = @schema_name 
          AND TABLE_NAME = 'contests' 
          AND COLUMN_NAME = 'scoring_mode'
    ),
    'ALTER TABLE contests ADD COLUMN scoring_mode VARCHAR(16) NOT NULL DEFAULT ''ACM'' ...',
    'SELECT 1'
);
PREPARE stmt FROM @sql; EXECUTE stmt; DEALLOCATE PREPARE stmt;
```

#### ✅ 重构判题模块安全架构

**问题**: `LocalJudgeService` 直接在主服务器执行用户代码，存在严重安全风险

**修复**: 
1. 添加配置开关 `qoj.judge.enable-local-judge` (默认 false)
2. 添加沙箱运行开关 `qoj.judge.enable-sandbox` (默认 false)
3. 更新配置文件和属性类

**新增配置**:
```yaml
# application.yml
qoj:
  judge:
    # 警告：本地判题直接在主服务器运行用户代码，存在严重安全风险
    # 生产环境务必设置为 false，使用 DOMjudge 进行判题
    enable-local-judge: ${ENABLE_LOCAL_JUDGE:false}
    # 是否允许沙箱运行（/api/v1/sandbox/run 接口）
    # 生产环境务必设置为 false
    enable-sandbox: ${ENABLE_SANDBOX:false}
```

**文件**: 
- `/backend/src/main/resources/application.yml`
- `/backend/src/main/java/com/qoj/config/QojProperties.java`

// 封榜逻辑：如果当前在封榜期间，非管理员和非创建者只能看到封榜时的榜单快照
if (contest.freezeTime != null && now.isAfter(contest.freezeTime) && now.isBefore(contest.endTime)) {

    if (!isPrivileged) {

    }
}

### 3.2 练习密码可被暴力破解
**位置**: `PracticeService.detail()`  
**问题**: 密码验证失败无重试次数限制  
**修复**: 添加 Redis 频率限制（5分钟内最多尝试5次）
```java
String rateKey = "practice:pwd:attempt:" + id + ":user:" +
    (CurrentUser.get() != null ? CurrentUser.get().id() : "anonymous");
Long attempts = redisTemplate.opsForValue().increment(rateKey);
if (attempts == 1) {
    redisTemplate.expire(rateKey, java.time.Duration.ofMinutes(5));
}
if (attempts > 5) {
    throw new BizException(429, "密码尝试次数过多，请5分钟后再试");
}
```
**文件**: `/backend/src/main/java/com/qoj/module/practice/service/PracticeService.java:112-120`
