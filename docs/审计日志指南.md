# 安全审计日志使用指南

## 概述

安全审计日志系统已集成到权限系统中，自动记录所有权限拒绝事件和敏感操作（删除、重判等）。

## 日志位置

审计日志保存在独立的文件中：
```
logs/security-audit.log              # 当前日志
logs/security-audit.2026-06-12.log   # 历史日志（按日期归档）
```

日志保留策略：
- 保留时间：90天
- 最大容量：50GB
- 自动按日期滚动

## 日志格式

每条审计日志包含以下信息：
```
时间戳 | 结果 | 用户信息 | 角色 | 资源信息 | 权限动作 | 拒绝原因
```

示例：
```
2026-06-13 11:30:45 | Permission DENIED | User: alice(123) | Role: STUDENT | Resource: Problem#456 | Action: VIEW | Reason: 未登录且题目非公开
2026-06-13 11:32:10 | Permission DENIED | User: bob(789) | Role: STUDENT | Resource: Contest#100 | Action: SUBMIT | Reason: 比赛未开始或已结束
2026-06-13 11:35:22 | Permission ALLOWED | User: admin(1) | Role: SUPER_ADMIN | Resource: Problem#456 | Action: DELETE | Reason: 超级管理员
```

## 常用查询命令

### 1. 查看所有权限拒绝事件
```bash
grep "DENIED" logs/security-audit.log
```

### 2. 查看特定用户的权限拒绝记录
```bash
grep "DENIED.*User: alice" logs/security-audit.log
```

### 3. 查看特定资源的访问记录
```bash
grep "Resource: Problem#456" logs/security-audit.log
```

### 4. 查看特定类型资源的拒绝事件
```bash
grep "DENIED.*Resource: Contest" logs/security-audit.log
```

### 5. 查看特定权限动作的拒绝记录
```bash
grep "DENIED.*Action: DELETE" logs/security-audit.log
```

### 6. 查看敏感操作记录（包含允许和拒绝）
```bash
grep -E "Action: (DELETE|REJUDGE)" logs/security-audit.log
```

### 7. 统计特定用户被拒绝的次数
```bash
grep "DENIED.*User: alice" logs/security-audit.log | wc -l
```

### 8. 查看最近1小时的拒绝事件
```bash
# 方法1：使用tail查看最新记录
tail -100 logs/security-audit.log | grep "DENIED"

# 方法2：使用时间过滤（需要根据实际时间调整）
grep "2026-06-13 11:" logs/security-audit.log | grep "DENIED"
```

### 9. 按拒绝原因统计
```bash
grep "DENIED" logs/security-audit.log | awk -F'Reason: ' '{print $2}' | sort | uniq -c | sort -rn
```

### 10. 查看特定日期的审计记录
```bash
# 查看昨天的日志
grep "DENIED" logs/security-audit.2026-06-12.log

# 查看最近7天的日志
for file in logs/security-audit.2026-06-*.log; do
    echo "=== $file ==="
    grep "DENIED" "$file" | wc -l
done
```

### 11. 导出为CSV格式
```bash
grep "Permission" logs/security-audit.log | \
  sed 's/ | /,/g' | \
  sed '1i时间戳,结果,用户信息,角色,资源信息,权限动作,拒绝原因' > audit-report.csv
```

### 12. 实时监控审计日志
```bash
# 实时查看所有审计日志
tail -f logs/security-audit.log

# 实时查看拒绝事件
tail -f logs/security-audit.log | grep --line-buffered "DENIED"
```

## 高级分析示例

### 分析异常访问模式
查找短时间内多次被拒绝的用户（可能是攻击行为）：
```bash
# 统计每个用户的拒绝次数
grep "DENIED" logs/security-audit.log | \
  awk -F'User: |\\(' '{print $2}' | \
  cut -d'(' -f1 | \
  sort | uniq -c | sort -rn | head -20
```

### 分析特定资源的访问热度
```bash
# 统计各类资源被拒绝的次数
grep "DENIED" logs/security-audit.log | \
  awk -F'Resource: ' '{print $2}' | \
  awk -F'#' '{print $1}' | \
  sort | uniq -c | sort -rn
```

### 生成每日安全报告
```bash
#!/bin/bash
# 生成今天的安全审计报告
TODAY=$(date +%Y-%m-%d)
LOG_FILE="logs/security-audit.log"

echo "=== 安全审计报告 - $TODAY ==="
echo ""

echo "1. 总体统计"
TOTAL=$(grep "$TODAY" "$LOG_FILE" | wc -l)
DENIED=$(grep "$TODAY.*DENIED" "$LOG_FILE" | wc -l)
ALLOWED=$(grep "$TODAY.*ALLOWED" "$LOG_FILE" | wc -l)
echo "   总记录数: $TOTAL"
echo "   拒绝次数: $DENIED"
echo "   允许次数: $ALLOWED"
echo ""

echo "2. Top 10 被拒绝的用户"
grep "$TODAY.*DENIED" "$LOG_FILE" | \
  awk -F'User: |\\(' '{print $2}' | \
  cut -d'(' -f1 | \
  sort | uniq -c | sort -rn | head -10
echo ""

echo "3. 拒绝原因分布"
grep "$TODAY.*DENIED" "$LOG_FILE" | \
  awk -F'Reason: ' '{print $2}' | \
  sort | uniq -c | sort -rn | head -10
echo ""

echo "4. 敏感操作记录"
grep "$TODAY.*Action: DELETE\|Action: REJUDGE" "$LOG_FILE"
```

## 集成到监控系统

### Logstash配置示例
```ruby
input {
  file {
    path => "/path/to/logs/security-audit.log"
    start_position => "beginning"
    codec => plain {
      charset => "UTF-8"
    }
  }
}

filter {
  grok {
    match => {
      "message" => "%{TIMESTAMP_ISO8601:timestamp} \| Permission %{WORD:result} \| User: %{DATA:username}\(%{NUMBER:user_id}\) \| Role: %{WORD:role} \| Resource: %{DATA:resource_type}#%{NUMBER:resource_id} \| Action: %{WORD:action} \| Reason: %{GREEDYDATA:reason}"
    }
  }
  
  date {
    match => ["timestamp", "yyyy-MM-dd HH:mm:ss"]
    target => "@timestamp"
  }
}

output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "qoj-security-audit-%{+YYYY.MM.dd}"
  }
}
```

## 安全建议

1. **定期审查**：建议每周审查一次审计日志，关注异常访问模式
2. **告警设置**：可以配置日志监控工具，当特定用户短时间内多次被拒绝时发送告警
3. **合规性**：审计日志保留90天，满足大多数合规要求
4. **访问控制**：限制审计日志文件的访问权限，仅允许管理员查看
5. **备份**：定期备份审计日志到安全的存储位置

## 常见问题

**Q: 为什么有些操作没有记录到审计日志？**
A: 审计日志主要记录权限拒绝事件和敏感操作（DELETE、REJUDGE）。正常的允许操作（如VIEW）不会记录，避免日志过大。

**Q: 审计日志会影响性能吗？**
A: 影响很小。日志采用异步写入，不会阻塞主业务流程。

**Q: 如何修改日志保留时间？**
A: 编辑 `backend/src/main/resources/logback-spring.xml`，修改 `SECURITY_AUDIT_FILE` appender的 `maxHistory` 参数。

**Q: 可以将审计日志发送到远程服务器吗？**
A: 可以。修改logback配置，添加远程appender（如Syslog、Socket等）。
