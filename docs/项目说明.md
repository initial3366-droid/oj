# QOJ 校园在线评测系统

QOJ (Qingling Online Judge) 是一个面向中学生的校园在线评测平台，支持题库、练习集、比赛（ACM/OI 双赛制）、排行榜等功能，提供完整的编程学习与竞赛体验。

## 项目介绍

**核心功能**:
- **题库系统**: 支持题目管理、分类、难度标签、样例数据、全文检索
- **练习集**: 教师可创建面向班级/社团的专项练习
- **比赛系统**: 支持 ACM（罚时）和 OI（分数）双赛制，含封榜、防切屏、代码查看权限控制
- **判题系统**: 集成 DOMjudge 远程判题，支持 Docker 沙箱隔离（开发中）
- **排行榜**: 全局/班级/社团三级 Rating 排名
- **用户管理**: 支持学生、教师、社团管理员、超级管理员四级权限
- **实时更新**: WebSocket 推送提交状态、比赛榜单、公告

**安全特性**:
- JWT 双 Token 轮换（Access Token 15 分钟，Refresh Token 7 天）
- 用户代码沙箱隔离（DOMjudge/Docker），禁止裸跑
- 细粒度权限控制（URL 层 + 角色层 + 资源层）
- 私有题目访问控制、比赛数据隔离、提交代码查看权限

**设计理念**:
- 前后端分离架构，RESTful API 设计
- 数据库驱动的动态配置（每日一题、轮播图、比赛规则）
- Flyway 版本化数据库迁移
- 响应式 UI，支持桌面端与移动端

## 技术栈

### 前端
- **框架**: React 19 + TypeScript 5
- **构建工具**: Vite 6
- **UI 组件**: NextUI (HeroUI)
- **路由**: React Router v6
- **代码编辑器**: Monaco Editor
- **数学公式**: KaTeX
- **WebSocket**: STOMP over SockJS

### 后端
- **框架**: Spring Boot 3.3.5 (Java 17)
- **安全**: Spring Security + JWT (HS512)
- **ORM**: MyBatis-Plus
- **数据库**: MySQL 8.0 (UTF8MB4)
- **缓存**: Redis 7
- **数据库迁移**: Flyway
- **WebSocket**: STOMP
- **API 文档**: SpringDoc OpenAPI 3

### 判题系统
- **远程判题**: DOMjudge API 集成
- **Docker 沙箱**: 隔离执行用户代码（开发中）
- **本地判题**: 仅供开发测试（默认禁用，不安全）

## 本地启动

### 前置要求
- Node.js 18+ / npm 9+
- Java 17+
- Maven 3.8+
- Docker & Docker Compose

### 快速启动（推荐）

**1. 克隆仓库**
```bash
git clone <repository-url>
cd qoj
```

**2. 启动数据库服务**
```bash
docker compose -f .runtime/qoj-deps.compose.yml up -d
```
等待 MySQL 和 Redis 健康检查通过（约 15 秒）。

**3. 启动后端**
```bash
cd backend
mvn spring-boot:run
```
后端将运行在 `http://127.0.0.1:8080`，Flyway 将自动执行数据库迁移。

**4. 启动前端**
```bash
npm install
npm run dev
```
前端将运行在 `http://127.0.0.1:5173`，自动代理 `/api` 和 `/ws` 请求到后端。

**5. 访问系统**
- **用户端**: http://127.0.0.1:5173
- **管理后台**: http://127.0.0.1:5173/admin
- **API 文档**: http://127.0.0.1:8080/swagger-ui.html

```bash
npm run build        # 构建到 dist/ 目录

```

### 配置说明
前端通过 Vite 环境变量配置后端地址：
- `VITE_API_URL`: API 基础 URL（默认 `/api`）
- `VITE_WS_URL`: WebSocket URL（默认 `http://127.0.0.1:8080/ws`）
- `VITE_API_PROXY_TARGET`: 开发代理目标（默认 `http://127.0.0.1:8080`）

创建 `.env.local` 文件覆盖默认配置：
```env
VITE_API_PROXY_TARGET=http://192.168.1.100:8080
```

## 数据库初始化

### 自动迁移（推荐）
QOJ 使用 Flyway 自动管理数据库版本：

1. **启动 MySQL 服务**
```bash
docker compose -f .runtime/qoj-deps.compose.yml up -d mysql
```

2. **首次启动后端，Flyway 自动创建表**
```bash
cd backend
mvn spring-boot:run
```

Flyway 将按顺序执行 `backend/src/main/resources/db/migration/` 中的 22 个迁移脚本：
- `V1__init_schema.sql`: 创建基础表
- `V2__club_invite_code.sql`: 添加社团邀请码
- `V3__seed_initial_data.sql`: 插入初始数据
- `V4__problem_test_cases.sql`: 创建测试用例表
- `V5__practices.sql`: 创建练习集表
- `V6__user_problem_status.sql`: 创建用户-题目状态缓存
- ...
- `V15__contest_rank_system.sql`: ACM/OI 榜单系统
- `V20__optimize_database_schema.sql`: 优化索引
- `V21__unify_submission_case_tables.sql`: 统一测试点表
- `V22__cleanup_deprecated_tables.sql`: 清理废弃表

### 手动初始化（不推荐）
如需手动执行，按顺序运行迁移脚本：
```bash
mysql -h127.0.0.1 -P13306 -uroot -proot qoj < backend/src/main/resources/db/migration/V1__init_schema.sql
mysql -h127.0.0.1 -P13306 -uroot -proot qoj < backend/src/main/resources/db/migration/V2__club_invite_code.sql
# ... 依次执行所有脚本
```

### 重置数据库
```bash
docker compose -f .runtime/qoj-deps.compose.yml down -v  # 删除容器和数据卷
docker compose -f .runtime/qoj-deps.compose.yml up -d    # 重新启动
cd backend && mvn spring-boot:run                         # Flyway 重新初始化
```

## 环境变量说明

QOJ 通过环境变量配置敏感信息。创建 `.env` 文件（不提交到 Git）：

```bash
# MySQL 配置
MYSQL_HOST=127.0.0.1
MYSQL_PORT=13306
MYSQL_DATABASE=qoj
MYSQL_USERNAME=root
MYSQL_PASSWORD=root

# Redis 配置
REDIS_HOST=127.0.0.1
REDIS_PORT=16379
REDIS_PASSWORD=

# JWT 配置（生产环境必须修改！）
JWT_SECRET=change-this-to-a-random-64-byte-string-for-production
JWT_ACCESS_EXPIRE=900      # Access Token 有效期（秒），默认 15 分钟
JWT_REFRESH_EXPIRE=604800  # Refresh Token 有效期（秒），默认 7 天

# DOMjudge 配置（可选，未配置时判题功能不可用）
DOMJUDGE_BASE_URL=http://domjudge.example.com
DOMJUDGE_API_KEY=your-api-key-here
DOMJUDGE_DEFAULT_CONTEST_ID=1

# 判题模式（domjudge | docker | unsafe-local）
JUDGE_MODE=domjudge

# ⚠️ 不安全的本地判题（仅供开发测试，生产环境禁止启用）
ENABLE_UNSAFE_LOCAL_JUDGE=false

# 沙箱运行（/api/v1/sandbox/run 接口）
ENABLE_SANDBOX=false

# CORS 跨域配置
CORS_ALLOWED_ORIGINS=http://127.0.0.1:5173,http://localhost:5173

# Spring Boot 配置
SPRING_PROFILES_ACTIVE=dev
SERVER_PORT=8080
```

### 重要说明
- **JWT_SECRET**: 生产环境必须使用 64 字节以上的随机字符串（HS512 算法要求）
- **ENABLE_UNSAFE_LOCAL_JUDGE**: 启用后用户代码将直接在后端服务器执行，存在严重安全风险，仅供本地开发调试
- **JUDGE_MODE**: 生产环境只能使用 `domjudge` 或 `docker`（开发中）

## 判题模块说明

QOJ 支持三种判题模式，优先级：**DOMjudge（推荐）> Docker（开发中）> 本地判题（不安全）**

### 1. DOMjudge 远程判题（推荐）

**原理**: 后端通过 DOMjudge API 提交代码，轮询获取判题结果，用户代码在 DOMjudge 的隔离环境中执行。

**配置**:
```yaml
qoj:
  judge:
    mode: domjudge
  domjudge:
    base-url: http://domjudge.example.com
    api-key: your-api-key
    contest-id: 1  # DOMjudge 比赛 ID
```

**优点**:
- ✅ 成熟稳定，生产环境验证
- ✅ 沙箱隔离，用户代码无法访问主服务器
- ✅ 支持多种编程语言和评测模式
- ✅ 自动处理超时、内存限制、输出限制

**部署 DOMjudge**:
```bash
# Docker 方式（快速测试）
docker run -d -p 8081:80 \
  --name domjudge \
  -e MYSQL_ROOT_PASSWORD=rootpw \
  domjudge/domjudge:latest

# 访问 http://127.0.0.1:8081，登录后获取 API Key
```

详见 DOMjudge 官方文档: https://www.domjudge.org/docs/

### 2. Docker 沙箱判题（开发中）

**原理**: 后端为每次提交创建临时 Docker 容器，在容器内编译并执行用户代码，限制资源使用。

**配置**:
```yaml
qoj:
  judge:
    mode: docker
    docker:
      image: qoj-judge:latest
      max-output-bytes: 10485760  # 10MB
      max-processes: 32
```

**状态**: 🚧 开发中，暂未实现

### 3. 本地判题（不安全，仅供开发）

**⚠️ 严重安全警告**: 用户代码直接在后端服务器进程中执行，可能导致：
- 任意文件读写（读取 `.env`、数据库配置）
- 进程劫持（`System.exit()` 终止服务）
- 资源耗尽（无限循环、内存泄漏）
- 网络攻击（内网扫描、反向 Shell）

**仅在以下情况启用**:
- 本地开发测试，无真实用户代码
- 无外网访问，仅 localhost 环境
- 临时验证判题流程

**启用方式**:
```yaml
qoj:
  judge:
    mode: unsafe-local
    enable-unsafe-local-judge: true
```

**代码实现**: `LocalJudgeService` 已标记 `@Deprecated`，通过 `@ConditionalOnProperty` 默认禁用。

## 常见问题

### 1. 后端启动失败：`Caused by: java.sql.SQLException: Access denied for user 'root'@'localhost'`

**原因**: MySQL 连接配置错误。

**解决**:
检查 `.env` 文件或环境变量：
```bash
export MYSQL_HOST=127.0.0.1
export MYSQL_PORT=13306
export MYSQL_USERNAME=root
export MYSQL_PASSWORD=root
```

验证 MySQL 连接：
```bash
docker exec -it qoj-mysql mysql -uroot -proot -e "SELECT 1"
```

### 2. 前端启动后 API 请求 404

**原因**: Vite 代理未生效或后端未启动。

**解决**:
1. 确认后端运行在 `http://127.0.0.1:8080`
2. 检查 `vite.config.ts` 代理配置：
   ```typescript
   proxy: {
     '/api': {
       target: 'http://127.0.0.1:8080',
       changeOrigin: true,
     },
   }
   ```
3. 浏览器控制台查看实际请求 URL

### 3. Flyway 迁移失败：`Validate failed: Detected resolved migration not applied`

**原因**: 数据库中已有旧版本迁移记录，与当前脚本不一致。

**解决**:
```bash
# 重置数据库（会丢失所有数据）
docker compose -f .runtime/qoj-deps.compose.yml down -v
docker compose -f .runtime/qoj-deps.compose.yml up -d
cd backend && mvn spring-boot:run
```

或手动清理 Flyway 历史：
```sql
DELETE FROM flyway_schema_history WHERE version > '20';
```

### 4. JWT 验证失败：`The Token's Signature resulted invalid`

**原因**: JWT_SECRET 长度不足 64 字节（HS512 算法要求）。

**解决**:
```bash
# 生成 64 字节随机字符串
openssl rand -base64 64

# 更新 .env
JWT_SECRET=<生成的随机字符串>
```

### 5. WebSocket 连接失败：`WebSocket connection to 'ws://127.0.0.1:5173/ws' failed`

**原因**: 前端 WebSocket URL 配置错误。

**解决**:
检查 `src/utils/websocket.ts`：
```typescript
const WS_URL = import.meta.env.VITE_WS_URL || "http://127.0.0.1:8080/ws";
```

创建 `.env.local`：
```env
VITE_WS_URL=http://127.0.0.1:8080/ws
```

### 6. 提交代码后一直显示 "判题中"

**原因**: 判题服务未配置或不可用。

**解决**:
1. 检查 `application.yml` 中的 `qoj.judge.mode`
2. 如果使用 DOMjudge，验证 API 连接：
   ```bash
   curl -H "Authorization: Bearer <API_KEY>" \
     http://domjudge.example.com/api/v4/contests
   ```
3. 查看后端日志：
   ```bash
   tail -f backend/logs/spring.log | grep JudgeCallbackService
   ```

### 7. 比赛榜单不更新

**原因**: 榜单缓存未刷新。

**解决**:
管理员登录后台，进入"比赛管理"→"重建榜单"，强制刷新缓存。

或调用 API：
```bash
curl -X POST http://127.0.0.1:8080/api/admin/v1/contests/{contestId}/rebuild-rank \
  -H "Authorization: Bearer <ADMIN_TOKEN>"
```

### 8. 前端构建失败：`Module not found: Error: Can't resolve '@heroui/react'`

**原因**: 依赖未安装或别名配置错误。

**解决**:
```bash
npm install
```

检查 `vite.config.ts` 别名配置：
```typescript
alias: {
  '@heroui/react': '@nextui-org/react',
}
```
