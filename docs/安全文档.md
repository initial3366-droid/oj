# QOJ 安全文档

本文档详细说明 QOJ 在线评测系统的安全机制、权限模型、代码执行隔离策略及数据保护措施。

---

## 1. 为什么不能裸跑用户代码

**核心原则**: 用户提交的代码不受信任，可能包含恶意代码，必须在隔离环境中执行。

### 1.1 安全风险

直接在主服务器执行用户代码会导致：

#### 风险 1: 任意文件读写
```java
// 恶意代码示例
import java.nio.file.*;
public class Main {
    public static void main(String[] args) throws Exception {
        // 读取数据库配置
        String env = Files.readString(Paths.get("/app/.env"));
        System.out.println(env);
        
        // 删除关键文件
        Files.delete(Paths.get("/app/backend/target/application.jar"));
    }
}
```

#### 风险 2: 进程劫持
```python
# 恶意代码示例
import os
os.system("pkill -9 java")  # 杀死后端进程
```

#### 风险 3: 资源耗尽
```cpp
// 恶意代码示例
#include <vector>
int main() {
    while (true) {
        std::vector<int> v(1000000000);  // 内存炸弹
    }
}
```

#### 风险 4: 网络攻击
```python
# 恶意代码示例
import socket
# 扫描内网
for i in range(1, 255):
    sock = socket.socket()
    sock.connect(('192.168.1.' + str(i), 22))
```

### 1.2 QOJ 的隔离方案

QOJ 支持三种判题模式，优先级：**DOMjudge > Docker > 本地判题（禁用）**

| 模式 | 隔离方式 | 安全性 | 生产可用 |
|------|---------|--------|---------|
| DOMjudge | 远程判题服务器 + chroot 沙箱 | ✅ 高 | ✅ 是 |
| Docker | 临时容器 + cgroup/namespace 隔离 | ✅ 高 | 🚧 开发中 |
| unsafe-local | 无隔离，直接执行 | ❌ 无 | ❌ 否 |

**生产环境配置**:
```yaml
qoj:
  judge:
    mode: domjudge  # 生产环境只能用 domjudge 或 docker
    enable-unsafe-local-judge: false  # 绝对禁止启用
```

## 2. Docker Judge 的限制

Docker 沙箱通过 Linux 内核特性隔离用户代码（开发中）。

### 2.1 资源限制

**CPU 限制**:
```bash
docker run --cpus="0.5" qoj-judge  # 限制 0.5 个 CPU 核心
```

**内存限制**:
```bash
docker run --memory="256m" --memory-swap="256m" qoj-judge
```

**磁盘 I/O 限制**:
```bash
docker run --device-read-bps=/dev/sda:1mb qoj-judge
```

**进程数限制**:
```bash
docker run --pids-limit=32 qoj-judge
```

### 2.2 网络隔离

**禁用网络访问**:
```bash
docker run --network none qoj-judge
```

用户代码无法：
- 访问外网（下载病毒、DDoS 攻击）
- 访问内网（扫描数据库、Redis）
- 建立反向 Shell

### 2.3 文件系统隔离

**只读根文件系统**:
```bash
docker run --read-only --tmpfs /tmp:rw,size=10m qoj-judge
```

用户代码无法：
- 修改系统文件
- 安装软件包
- 持久化恶意代码

### 2.4 能力限制（Capabilities）

Docker 默认移除危险的 Linux Capabilities：

```bash
docker run --cap-drop=ALL --cap-add=SETGID --cap-add=SETUID qoj-judge
```

禁止的操作：
- `CAP_SYS_ADMIN`: 挂载文件系统、修改系统参数
- `CAP_NET_ADMIN`: 修改网络配置
- `CAP_SYS_MODULE`: 加载内核模块

### 2.5 Seccomp 系统调用过滤

禁止危险的系统调用：

```json
{
  "defaultAction": "SCMP_ACT_ERRNO",
  "syscalls": [
    {"names": ["clone", "fork"], "action": "SCMP_ACT_ALLOW"},
    {"names": ["execve"], "action": "SCMP_ACT_ERRNO"}
  ]
}
```

禁止的操作：
- `execve`: 执行外部程序
- `ptrace`: 调试其他进程
- `reboot`: 重启系统

### 2.6 运行时限制

**超时自动终止**:
```java
// JudgeService.java
CompletableFuture.runAsync(() -> {
    dockerClient.exec(container, code, input);
}).orTimeout(timeLimit, TimeUnit.MILLISECONDS);
```

**输出大小限制**:
```yaml
qoj:
  judge:
    docker:
      max-output-bytes: 10485760  # 10MB
```

防止输出炸弹（无限输出耗尽磁盘）。

## 3. JWT 安全

### 3.1 双 Token 设计

**Access Token**:
- 有效期短（默认 15 分钟）
- 用于 API 请求认证
- 泄露风险窗口小

**Refresh Token**:
- 有效期长（默认 7 天）
- 仅用于刷新 Access Token
- 支持主动撤销

### 3.2 Token 轮换（Refresh Token Family）

每次刷新 Token 时，旧 Refresh Token 立即失效：

```java
// JwtService.java
public TokenPair refreshToken(String oldRefreshToken) {
    String familyId = extractFamilyId(oldRefreshToken);
    
    // 检测重放攻击
    if (refreshTokenBlacklist.contains(oldRefreshToken)) {
        // 同一 familyId 的所有 Token 全部失效
        revokeTokenFamily(familyId);
        throw new SecurityException("检测到 Token 重放攻击");
    }
    
    // 旧 Token 加入黑名单
    refreshTokenBlacklist.add(oldRefreshToken);
    
    // 生成新 Token（相同 familyId）
    return generateTokenPair(userId, familyId);
}
```

**防御场景**:
- 攻击者窃取 Refresh Token A
- 用户正常刷新：A → B（A 失效）
- 攻击者使用 A 刷新：检测到重放，B 也失效
- 用户被强制重新登录，攻击者的 Token 无效

### 3.3 Token 黑名单

使用 Redis 存储已失效的 Token：

```java
// JwtBlacklistService.java
public void addToBlacklist(String token) {
    long ttl = extractExpiration(token).getTime() - System.currentTimeMillis();
    redisTemplate.opsForValue().set(
        "jwt:blacklist:" + token,
        "revoked",
        ttl,
        TimeUnit.MILLISECONDS
    );
}
```

**应用场景**:
- 用户主动登出
- 管理员封禁用户
- 检测到 Token 泄露

### 3.4 签名算法

使用 HS512（HMAC-SHA512）签名：

```java
// JwtService.java
private Key getSigningKey() {
    byte[] keyBytes = jwtSecret.getBytes(StandardCharsets.UTF_8);
    if (keyBytes.length < 64) {
        throw new IllegalStateException("JWT_SECRET 必须至少 64 字节");
    }
    return Keys.hmacShaKeyFor(keyBytes);
}

public String generateToken(Long userId) {
    return Jwts.builder()
        .setSubject(userId.toString())
        .signWith(getSigningKey(), SignatureAlgorithm.HS512)
        .compact();
}
```

**安全要求**:
- `JWT_SECRET` 长度 ≥ 64 字节
- 生产环境必须使用强随机字符串
- 禁止硬编码在代码中

## 4. 权限模型

QOJ 使用三层权限控制：**URL 层 → 角色层 → 资源层**

### 4.1 URL 层权限（SecurityConfig）

Spring Security 配置 URL 访问规则：

```java
// SecurityConfig.java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) {
    http.authorizeHttpRequests(auth -> auth
        // 管理端用户管理：仅超级管理员
        .requestMatchers("/api/admin/v1/users/**").hasRole("SUPER_ADMIN")
        
        // 其他管理端接口：教师、社团管理员、超级管理员
        .requestMatchers("/api/admin/v1/**").hasAnyRole(
            "SUPER_ADMIN", "TEACHER", "CLUB_ADMIN"
        )
        
        // 用户端写操作：需登录
        .requestMatchers(POST, "/api/v1/**").authenticated()
        .requestMatchers(PUT, "/api/v1/**").authenticated()
        .requestMatchers(DELETE, "/api/v1/**").authenticated()
        
        // 用户端读操作：公开
        .requestMatchers(GET, "/api/v1/problems/**").permitAll()
        .requestMatchers(GET, "/api/v1/contests/**").permitAll()
    );
    
    return http.build();
}
```

### 4.2 角色层权限（@PreAuthorize）

Controller 方法级别的角色检查：

```java
// AdminUserController.java
@PreAuthorize("hasRole('SUPER_ADMIN')")
@PostMapping
public ApiResponse<UserVO> createUser(@RequestBody CreateUserRequest request) {
    // 只有超级管理员可以创建用户
}

// AdminProblemController.java
@PreAuthorize("hasAnyRole('SUPER_ADMIN', 'TEACHER')")
@PostMapping
public ApiResponse<ProblemVO> createProblem(@RequestBody CreateProblemRequest request) {
    // 超级管理员和教师可以创建题目
}
```

### 4.3 资源层权限（AccessPolicy）

业务逻辑层的细粒度权限控制，检查用户与资源的所有权关系。

#### 4.3.1 题目访问权限

```java
// ProblemAccessPolicy.java
@Component
public class ProblemAccessPolicy {
    
    public boolean canView(Problem problem, User user) {
        // 公开题目：所有人可见
        if (problem.getIsPublic()) {
            return true;
        }
        
        // 私有题目：仅所有者和超级管理员可见
        if (user == null) {
            return false;
        }
        
        return user.getRole() == UserRole.SUPER_ADMIN
            || problem.getOwnerId().equals(user.getId());
    }
    
    public boolean canEdit(Problem problem, User user) {
        if (user == null) {
            return false;
        }
        
        // 超级管理员可编辑所有题目
        if (user.getRole() == UserRole.SUPER_ADMIN) {
            return true;
        }
        
        // 题目所有者可编辑自己的题目
        return problem.getOwnerId().equals(user.getId());
    }
}
```

#### 4.3.2 比赛访问权限

```java
// ContestAccessPolicy.java
@Component
public class ContestAccessPolicy {
    
    public boolean canViewProblems(Contest contest, User user) {
        // 比赛开始后，参赛者可见题目
        if (contest.getStatus() == ContestStatus.RUNNING 
            || contest.getStatus() == ContestStatus.ENDED) {
            return isParticipant(contest, user);
        }
        
        // 比赛前，仅所有者和管理员可见
        return isOwnerOrAdmin(contest, user);
    }
    
    public boolean canSubmit(Contest contest, User user) {
        // 比赛进行中
        if (contest.getStatus() != ContestStatus.RUNNING) {
            return false;
        }
        
        // 必须是参赛者
        if (!isParticipant(contest, user)) {
            return false;
        }
        
        // 管理员和所有者不能提交（避免作弊）
        if (isOwnerOrAdmin(contest, user)) {
            return false;
        }
        
        return true;
    }
}
```

#### 4.3.3 提交代码查看权限

```java
// SubmissionAccessPolicy.java
@Component
public class SubmissionAccessPolicy {
    
    public boolean canViewCode(Submission submission, User user) {
        if (user == null) {
            return false;
        }
        
        // 超级管理员可查看所有代码
        if (user.getRole() == UserRole.SUPER_ADMIN) {
            return true;
        }
        
        // 提交者本人可查看自己的代码
        if (submission.getUserId().equals(user.getId())) {
            return true;
        }
        
        // 其他人（包括教师、社团管理员）不能查看
        return false;
    }
    
    public boolean canRejudge(Submission submission, User user) {
        // 仅超级管理员可重判
        return user != null && user.getRole() == UserRole.SUPER_ADMIN;
    }
}
```

## 5. 私有题目防泄露

### 5.1 访问控制

私有题目（`is_public = false`）仅对以下用户可见：
- 题目所有者（创建者）
- 超级管理员

**实现**:
```java
// ProblemService.java
public Problem getProblemById(Long id, User currentUser) {
    Problem problem = problemMapper.selectById(id);
    
    if (!problemAccessPolicy.canView(problem, currentUser)) {
        throw new ForbiddenException("无权访问该题目");
    }
    
    return problem;
}
```

### 5.2 搜索过滤

题目搜索和列表接口自动过滤私有题目：

```java
// ProblemService.java
public PageResult<Problem> listProblems(User currentUser) {
    LambdaQueryWrapper<Problem> query = new LambdaQueryWrapper<>();
    
    if (currentUser == null || currentUser.getRole() != UserRole.SUPER_ADMIN) {
        // 非管理员只能看到公开题目或自己的题目
        query.and(w -> w.eq(Problem::getIsPublic, true)
            .or()
            .eq(Problem::getOwnerId, currentUser.getId())
        );
    }
    
    return problemMapper.selectPage(query);
}
```

### 5.3 比赛题目快照

比赛使用的题目会创建快照副本，避免题目修改影响比赛：

```java
// ContestService.java
public void addProblemToContest(Long contestId, Long problemId) {
    Problem problem = problemService.getProblemById(problemId);
    
    // 创建题目快照（包括题面、测试数据）
    ContestProblem snapshot = ContestProblem.builder()
        .contestId(contestId)
        .problemId(problemId)
        .title(problem.getTitle())
        .statement(problem.getStatement())
        .sampleCases(problem.getSampleCases())
        .build();
    
    contestProblemMapper.insert(snapshot);
}
```

## 6. 比赛数据隔离

### 6.1 参赛者身份隔离

用户可以多种身份参加比赛：
- **个人身份** (PERSONAL)
- **班级身份** (CLASS)
- **社团身份** (CLUB)

**数据模型**:
```java
// ContestParticipant.java
@Data
public class ContestParticipant {
    private Long id;
    private Long contestId;
    private Long userId;
    private IdentityType identityType;  // PERSONAL/CLASS/CLUB
    private Long identityId;            // 班级ID/社团ID
    private String nickname;
}
```

**隔离效果**:
- 班级比赛的榜单只显示班级成员
- 社团比赛的榜单只显示社团成员
- 不同身份的提交记录分开统计

### 6.2 榜单权限控制

```java
// ScoreboardAccessPolicy.java
@Component
public class ScoreboardAccessPolicy {
    
    public boolean canViewScoreboard(Contest contest, User user) {
        // 公开榜单：所有人可见
        if (contest.getScoreboardVisibility() == Visibility.PUBLIC) {
            return true;
        }
        
        // 参赛者榜单：仅参赛者可见
        if (contest.getScoreboardVisibility() == Visibility.PARTICIPANT) {
            return contestService.isParticipant(contest, user);
        }
        
        // 管理员榜单：仅管理员可见
        if (contest.getScoreboardVisibility() == Visibility.ADMIN) {
            return contest.getOwnerId().equals(user.getId())
                || user.getRole() == UserRole.SUPER_ADMIN;
        }
        
        return false;
    }
}
```

### 6.3 封榜机制

比赛榜单支持封榜（冻结排名），封榜后的提交结果不公开：

```java
// Contest.java
public boolean isFrozen() {
    if (freezeTime == null) {
        return false;
    }
    
    LocalDateTime now = LocalDateTime.now();
    return now.isAfter(freezeTime) && now.isBefore(endTime);
}

// ContestRankService.java
public RankResponse getContestRank(Long contestId, boolean showFrozen) {
    Contest contest = contestMapper.selectById(contestId);
    
    if (contest.isFrozen() && !showFrozen) {
        // 返回封榜时刻的快照
        return getScoreboardSnapshot(contestId, SnapshotType.FROZEN);
    }
    
    // 返回实时榜单
    return buildRealTimeRank(contestId);
}
```

### 6.4 防切屏监控

比赛可启用防切屏功能，记录参赛者的切屏行为：

```java

@Data
public class TabSwitchLog {

    private Long contestId;

public void recordTabSwitch(Long contestId, Long userId) {

}
```

## 7. 提交代码查看权限

### 7.1 查看规则

**默认规则**:
- ✅ 提交者本人可查看自己的代码
- ✅ 超级管理员可查看所有代码
- ❌ 其他用户（包括教师、社团管理员）不能查看

**实现**:
```java
// SubmissionController.java
@GetMapping("/{id}/code")
public ApiResponse<String> getSubmissionCode(
    @PathVariable Long id,
    @CurrentUser User currentUser
) {
    Submission submission = submissionService.getById(id);
    
    if (!submissionAccessPolicy.canViewCode(submission, currentUser)) {
        throw new ForbiddenException("无权查看该提交代码");
    }
    
    return ApiResponse.success(submission.getCode());
}
```

### 7.2 比赛代码查看

比赛结束后，根据比赛的 `code_visibility` 设置决定代码可见性：

```java
// Contest.java
public enum CodeVisibility {
    NEVER,          // 永不公开
    AFTER_CONTEST,  // 比赛结束后公开
    ALWAYS          // 始终公开
}

// SubmissionAccessPolicy.java
public boolean canViewCode(Submission submission, User user) {
    // 非比赛提交：按默认规则
    if (submission.getContestId() == null) {
        return isOwnerOrAdmin(submission, user);
    }
    
    // 比赛提交：检查比赛设置
    Contest contest = contestMapper.selectById(submission.getContestId());
    
    if (contest.getCodeVisibility() == CodeVisibility.NEVER) {
        return isOwnerOrAdmin(submission, user);
    }
    
    if (contest.getCodeVisibility() == CodeVisibility.AFTER_CONTEST) {
        return contest.getStatus() == ContestStatus.ENDED
            && contestService.isParticipant(contest, user);
    }
    
    if (contest.getCodeVisibility() == CodeVisibility.ALWAYS) {
        return contestService.isParticipant(contest, user);
    }
    
    return false;
}
```

## 安全检查清单

### 部署前必检项

- [ ] `JWT_SECRET` 已修改为 64 字节以上的随机字符串
- [ ] `ENABLE_UNSAFE_LOCAL_JUDGE` 设置为 `false`
- [ ] `JUDGE_MODE` 设置为 `domjudge` 或 `docker`
- [ ] MySQL、Redis 使用强密码
- [ ] CORS 配置仅允许可信域名
- [ ] `.env` 文件未提交到 Git
- [ ] 生产环境关闭 Swagger UI（`springdoc.swagger-ui.enabled=false`）
- [ ] 启用 HTTPS（使用反向代理如 Nginx）

### 定期检查项

- [ ] 审查审计日志，检测异常操作
- [ ] 检查 Token 黑名单大小，防止 Redis 内存溢出
- [ ] 更新依赖库，修复已知安全漏洞
- [ ] 备份数据库，制定灾难恢复计划
