# 安全审查报告

## 严重问题 (Critical)

### 1. ⚠️ 用户代码直接在主服务器执行
**文件**: `LocalJudgeService.java`
**风险**: 用户可以提交恶意代码，直接在主服务器上执行，可能导致：
- 服务器资源耗尽
- 敏感文件读取
- 反向shell攻击
- 数据库密码泄露

**当前实现**:
```java
Process process = new ProcessBuilder(command).directory(workDir.toFile()).start();
```

**建议**:
1. 使用 DOMjudge 进行远程判题（项目已配置）
2. 如果必须本地判题，需要：
   - 使用 Docker/沙箱隔离
   - 限制系统调用 (seccomp)
   - 禁止网络访问
   - 限制文件访问
   - 使用独立的判题用户（非 root）

### 2. ⚠️ 新增排名 API 缺少权限控制
**文件**: `ContestRankController.java`
**问题**: 所有排名 API（包括管理员操作）都没有权限注解

**当前**:
```java
@PostMapping("/{contestId}/rank/acm/rebuild")  // 缺少 @PreAuthorize
public ApiResponse<Void> rebuildAcmRank(@PathVariable Long contestId)
```

**影响**:
- 任何人都可以重建排名（可能导致性能问题）
- 任何人都可以创建/删除快照

### 3. ⚠️ GET 请求允许匿名访问排名 API
**文件**: `SecurityConfig.java`
**问题**: 
```java
.requestMatchers(HttpMethod.GET, "/api/v1/contests/**").permitAll()
```
这会让 `/api/contests/{id}/rank/*` 也允许匿名访问，但路径不匹配（应该是 `/api/v1/contests`）

## 中等问题 (Medium)

### 4. Admin 接口缺少细粒度权限
**问题**: 多个 Admin Controller 没有方法级权限注解
- `AdminHomeController.java`
- `AdminOrganizationController.java`
- `AdminContestController.java`

**建议**: 添加 `@PreAuthorize("hasAnyRole('SUPER_ADMIN', 'TEACHER', 'CLUB_ADMIN')")`

### 5. 临时文件清理不完整
**文件**: `LocalJudgeService.java`
**问题**: 如果判题过程中抛异常，临时文件可能不被删除

**当前**:
```java
try {
    workDir = Files.createTempDirectory("qoj-judge-");
    // ...
} finally {
    deleteDir(workDir);  // 但 deleteDir 捕获了所有异常
}
```
